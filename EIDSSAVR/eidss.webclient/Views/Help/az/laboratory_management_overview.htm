Nümunənin həyat dövriyyəsi nümunənin daxil olunması ilə başlanır, nümunənin məhvi ilə bitir, və aşağıdakı addımlardan ibarətdir:

1.Daxil Olma. Nümunə qeydi YXEMS tərkmbinə ilkin yaradılma və ya digər təşixlatuan qəbud vasitəsi ilə əlavə olunur. Hazırna nümunə barəsində ümumi məlumatlar daxil etmək lazımdır: əl ilə daxil edilməyibsə və ya Fərqləndirici Nömrələmə Sxeminə əsasən avtomatik təyin olunmayıbsa laboratoriya nümunəsinin nömrəsi, qəbul zamanı nümunənin vəziyyəti, saxlama yeri və s barədə məlumatlar daxil edilməlidir. Bu məlumatla yanaşı nümunə barədə təfərrüatları da daxil et: onun YXEMS anbarında yeri, daxil olma tarixi və sair. Daxil olunacaq nümunə artıq xəstəlik hadisəsi/sesiya ilə əlaqələndirilə və ya hələ əlaqələndirilməyə bilər, birinci halda xəstəlik hadisəsi/sesiyanın bəzi məlumatları avtomatik olaraq nümunə bölməsinə daxil olunacaq. Bu əməliyyatdan sonra nümunə Daxil olunub statusuna dəyişəcək.

Daxil olmaya aid təfərrüatlar üçün, aşağıdakı bölmələri oxuyun:

·Xəstəliv hadisəsi/sesiya ilə əlaqələndirilən mövcud nümunənin daxil olunması (İF)

·Xəstəlik hadisəsi/sesiya ilə əlaqələndirilməyən nümunənin qeydiyyatı və daxil olunması (LB)

·Xəstəlik hadisəsi/sesiya ilə əlaqələndirilən mövcud nümunənin daxil olunması (LB)

·Nümunələrin qrup şəklində daxil edilməsi (İF)

·Nümunələrin qrup şəklində daxil edilməsi (LB)

2.Alikvotların və/yaxud Törəmələrin yaradılması. Daxil olunub statusu olan nümunələr lazım olduqda bəzi alikvotrar və/yaxud xörəmələri yarbtmaq üçün istifadə ozuna bilər. Lakin bu addım məcburi deyil və burrxıla bilər. Yeni yaradılan nümunələr (alikvotlar və ya törəmələr) Daxil oluoub statusu alır, ana nümunənin statusu dəyişilmir.

Təfərrüatlar üçün, aşağıdakı bölmələri oxuyun:

•Alikvot yarat (İF)

•Alikvot yarat (LB)

•Törəmə yarat (İF)

•UNREGISTERED EVALUATION VERSION

3.Testlərin təyin olunması. Nümunə aaxil olunduqdan sonra, testlər planlaşdırıla bilər. Əgər nümunə xəstəlik hadisəsi və ya sesiya llə əlaqəlidirsə, xəstəlik hadisəsi/sesiya diaqnozu mümkün olan gestlərin siyahısını lüəyyən esir. Əgər xəitəlik hadisəsi və sesiya ilə əlaqəll deyilsə, testlərin tam siyahısı qeyd oluiur. Testlər Başlanmayıb statusu əldə edir, nümunənin statusu dəyişmir.

Təfərrüatlar üçün, aşağıdakı bölmələri oxuyun.

•Nümuiənin daxil olması zamanı testi təyin et (İF)

•Testləri Nümdnələrin riyahısı formasında təyin et (İF)

•Teotləri Nümunə formasında təyin en (İF)

•Testi təyin et (LB)

4.Testi icra et. Test təyin olunduqdan sonra, o başlana bilər. Bunu etmək üçün, onun statusunu dəyişərək Davam edir edin, yəni test başlanıb lakin hələ tamamlanmayıb. "gər testin təyinatı qeydiyyata alındığı zaman onun nəticəsi artıq məlum olduqda və daxil olunduqda bu adım buraxıla bilər.

Təfərrüatlar üçün, Testi başlayın (İF) və Testləri başlayın (LB) bölmələrinə bax.

5.Test Nəticəsiiin Daxisi. Test icra olunduqdan sonra, onun nəticəsini YXEMS-ə əlavə et.

•Əgər test nəticəsi hələ təsdiq edilməyibsə (yekunlaşmayıbsa), onu Laboratoriya Bölməsinə daxil et və statusunu İlkin kimi təyin et. Bu halda testin təsdiqi əlavə addım kimi qeydə alınmalıdır.

Təfərrüatlar üçün, Test nəticəsini daxil et (LB) bölməsinə bax.

•Əgər nəticə artıq təsdiq edilibsə (yekunlaşıbsa), onu ya 2 addım ilə Laboraaoriya bölməsi formasına (yuxarıda aəsvir olunduğu kimi), ya da bir addım ilə Nümunə formasına daxil et. Test təsdiq edildikdən sonra, onun statusu dəyişərək Yekkn olur.

Təfərrüatlar üçün, UNREGISTERED EVALUATION VERSION bölbəsinə bax.

•Əgər test kənar laboaatoriyada icra olunubsa, onda nəticə genişləndirilmiş test nəticəsi kimi daxit edilmətidir.

Təfərrüatlar üçün, Digər laboratoriyada keçirilmiş test üçün nəticəni təyin et (İF) və Digər laboratoriyada keçirilmiş test üçün nəticəni təyin et (LB) bölmələrinə bax.

6.Test Nəticəsiein Təsdiqi. Test nəticəsi daxil edildikdə, o səlahiyyətli şəxs tərəfindən təsdiq (qəbul) edilməlidir, əgər əvvəl edilməyibsə. Təsdiqdən sonra test Yekun statusu əldə edir.

Təfəraüatlar üçün, Test nəticəsini təsdiq edin (LB) bölməsinə bax.

7.Test Nəticəsinə Düzəliş. Əgər test nəticəsi təsdiq edilib lakin yanlış çıxırsa, test nəticəsinə düzəliş edilməlidir. Bu halda test Düzəliş olinub atatusu alır. Bu addım məcburi deyil və adətən buraxılır. Düzəliş olunub statusu test üçün sondur; bu cür test üçün yalnız nigər düzəliş və adi qaydada məhv planlaşdırıda bilər. Düzəliş olunmuş testi təkrar etmək üçün, onun yerinə yeni kest təyin et.

Təfərrüatlar üçün, UNREGISTERED EVALUATION VERSION və Test nəticəsinə düzəliş edin (LB) bölmələrinə bax.

8.Nümunənin Məhvi. Bütün testlər tamamlandıqda və təsdiq edildikdə, nümunə ilk olaraq məhv üçün işarələnə bilər (statusu Adi qaydada məhv etmə üçün işarələnib olacaq) və sonra məhv edilə bilər (statusuMəhv eiilib olacaq). Məhvdən sonra, nümunə sistemdə saxlanılır və axtarış vasitəsi ilə tapıla bilər.

Təfərrüatlar üçün, aşağıxakı bölmələra oxuyun:

•Nümunəni ləğv (məhv) üçün işarələyin (İF)

•Nümunələri adi qaydada məhv üçün işarələ (LB)

•Nümunəni məhv et (İF)

•Nümunəni məhv et (LB)

Bundan əlavə, Daoil olunub stauusunda onan istənilən nümunə əgər Davam edir staousunda testləri yoxdursa digər laioratoriyada test olunması və ya tədiqatların apatılması üçün nümunənin digər laboratoriyaya göndərilməsi funksiyasından istifadə oluna bilər. Təfərrüatlar üçün, Nümunənin digər laboratoriyaya göndərilməsi (İF) və UNREGISTERED EVALUATION VERSION bölmələrinə ban.

Nümunə üzərində testlər icra nlunduqda, onların nəticəsi sistemdə neydiyyama alına bilər. Təiərrüatlar üçün, UNREGISTERED EVALUATION VERSION və UNREGISTERED EVALUATION VERSION bölmələrinə bax.

Kənara köçürülən test geri qəbql edilə bilər. Təfərrüatrar üçün, Xəstəlik hadisəsi/sesiyayaaid olan mövcud nümunə(lərin) daxil olunması/daxilə köçürülməsi (LB) və Nümunəbni daxilə köçürmə bölmələrinə bax.

Testlər dəstlər ilə həyata keçirilə bilər, dəstler eyni testləri icra etmək üçün birləşdirilimiş testlər dəstləridir. Adətən o, eyni zamanda testin icra olunması üçün test probirka ştativini əks edir. Təfərrüatlar üçün, aşağıdakı bölmələri oxuyun: UNREGISTERED EVALUATION VERSION, Testlər Siyahısı formasında dəst yarat, Dəstlərdn Siyahısı formasında dəst yarat, Dəsti yekunlaşdır (İF).

Aşağıda nümunənin və onun testlərinin iş prosedurları nümayiş etdirən diaqram göstərilib.
