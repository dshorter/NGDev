Форма L05 «Приём в систему случая заболевания человека»

Вид формы L05 Приём в систему зависит от типа случая заболевания или сессии, относящихся к пробе. Фактически, разделы Сведения о пробе и Дополнительно запрошенные тесты и примечания по пробам являются аналогичными и описываются в разделе формы L05 Приём в систему. Различаются только разделы Общая информация и Получение / регистрация проб. Описание их вида представления пробы, связанной со случаем заболевания человека приведено ниже.

Рис. 345 Приём в систему

Рис. 346 Раздел общей информации

Поля панели Общая информация недоступны, за исключением поля Идентификатор случая заболевания, в котором можно выбрать другой случай заболевания для перезагрузки всей формы.

Доступность полей панели Получение / регистрация проб зависит от статуса пробы. У вновь созданной записи о пробе все поля доступны, за исключением поля Тесты, но только до сохранения формы. После первого сохранения записи, поля переходят в режим только для чтения.

У существующей записи о пробе, во время процесса приёма в систему, следующие поля являются редактируемыми: Лабораторный идентификатор пробы, Дата приёма в систему, Состояние пробы при получении, Комментарий, Место хранения и Обрабатывающий отдел. Поля доступны до первого сохранения формы; после него возможность их редактирования исчезает.

Все поля проб со статусом Принята в систему закрыты для редактирования.

 

Поля общей информации

Идентификатор случая заболевания

Является уникальным идентификатором случая заболевания. Для открытия другого случая заболевания,

1.нажмите кнопку Найти (). Откроется форма H02 Сообщение о случае заболевания человека.

2.Либо отметьте необходимый случай заболевания в таблице и нажмите кнопку Выбрать, либо дважды нажмите по нему мышкой.

После выбора случая заболевания, форма поиска закроется. Информация о случае заболевания автоматически заполнит поля панели Общая информация.

Для просмотра сведений о случае заболевания, нажмите кнопку Сведения (). Откроется форма H02 Сообщение о случае заболевания человека.

Ф.И.О. пациента

Содержит фамилию, имя и отчество пациента.

Возраст

Возраст пациента.

Дата появления симптомов

Указывает дату первого появления симптомов.

Диагноз

Диагноз, поставленный случаю заболевания.

Поля «Получение / регистрация проб»

Тип пробы

Указывается тип для новой пробы. Тип существующей пробы изменить невозможно.

Тесты

Отображается общее количество тестов, назначенных для пробы.

Локальный идентификатор пробы

Указывается локальный идентификатор новой пробы. Существующие в системе пробы уже имеют назначенные локальные идентификаторы пробы и изменить их невозможно.

Лабораторный идентификатор пробы

Лабораторный идентификатор пробы для создаваемой новой пробы генерируется автоматически и может быть изменён только до первого сохранения записи. У существующих в системе проб это поле является нередактируемым.

Дата взятия

Указывается дата взятия пробы. Редактирование поля возможно только во время создания записи; у записей о принятых в систему пробах это поле является нередактируемым.

Дата приёма в систему

Используется для указания даты приёма пробы в систему. По умолчанию используется значение текущей даты, которое можно изменять. После приёма пробы в систему, поле становится нередактируемым.

Состояние пробы при получении

В поле указывается состояние пробы во время её приёма в систему. Если состояние пробы является любым кроме хорошего, в столбец Комментарий потребуется вписать пояснение.

После сохранения пробы, поле становится нередактируемым.

Комментарий

В нём отображаются комментарии. В отношении отклонённой пробы или пробы, принятой в систему в плохом состоянии, поле является обязательным для заполнения текстом пояснения.

Место хранения

Указывается место хранения свежепринятой в систему пробы. Поле доступно для редактирования до первого сохранения формы после приёма пробы в систему.

Обрабатывающий отдел

Указывается отдел той организации, в которой проба хранится и анализируется. Это относится только к принятым в систему пробам со статусом в хорошем или плохом состоянии.

Кнопки

Поиск

Служит для поиска и добавления списка проб путём заполнения их локального или полевого идентификатора, который вводят в поле Поиск по локальному / полевому идентификатору пробы.

1.Для начала поиска, укажите локальный или полевой идентификатор пробы в поле Поиск по локальному / полевому идентификатору пробы.

2.Нажмите кнопку Найти. В список найденного будут добавлены все пробы, имеющие введённый локальный / полевой идентификатор пробы.

Печать штрихкодовых наклеек

Вызывает меню печати штрихкодовых наклеек принятых в систему проб. Печатать можно только пробы со статусом принятых в хорошем или плохом состоянии.

1.Сохраните форму.

2.Выберите уже принятую в систему (не отклонённую) пробу.

3.Нажмите кнопку Печать штрихкодовых наклеек. Откроется форма предварительного просмотра Печать штрихкодовых наклеек.

4.Нажмите кнопку Печать ().

Приём в систему

Используется для приёма в систему выбранных проб. Принять в систему возможно только ещё не принятые пробы.

1.Выберите не принятую пробу в таблице.

2.Нажмите кнопку Приём в систему. Выбранная запись станет доступной для редактирования.

Новая

Нажатие создаёт запись о новой пробе. Для получения более подробной информации, прочитайте раздел Приём в систему пробы, связанной или нет со случаем заболевания/сессией (ОФ).УдалениеУдаляет отмеченные пробы. Удалить можно только те записи, которые были добавлены посредством настоящей формы Приём в систему, и которые ещё не были сохранены. Для удаления пробы, отметьте её и нажмите кнопку Удалить.Выбор теста Служит для назначения теста отмеченной пробе. Для получения более подробной информации, прочитайте раздел Назначение тестов во время приёма пробы в систему (ОФ).

1.Нажмите мышкой на пробе в списке.

2.Нажмите кнопку Выбрать тест. Откроется форма L20 Выбор теста.
