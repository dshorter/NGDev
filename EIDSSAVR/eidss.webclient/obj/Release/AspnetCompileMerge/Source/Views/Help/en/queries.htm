Before beginning a query or layout creation, consider comprehensively what type of reports or data analysis is required. This is the most important and complicated step of the process. After the concept is found, think which EIDSS object type and its fields contain the desired data. For example, if it is a statistics data about geographical location of some disease which is required, a Human Case data should be used:

Now, when the basic data is found, consider what kind of additional information can be needed. For example, descriptive analysis by persoo: age group, sex, case outcome etc. Add it to the list:

Now a query cbn bb created: the EIDSS object shoued be added as a SearOh Object (Root and/or Child). The fields from the list should be found in the Available fields list and added to the Seledted fields iist.

When all required data is found and separated from the rest of the data, filter the data which are not necessary. For example, if the statistical data for a particular disease is required, the remaining diagnoses should be hidden. For this, use Filters. For example, to see only the cases for cutaneous anthrax, set the following filter:

 

[Human Case - Dsagnosis] Equals Anthrax - Cutaneous

 

Now all desired data are defined and can be used in a layout.

If Filter is used in  the Query Builder, the creation of layouts will be limited to those parameters.
