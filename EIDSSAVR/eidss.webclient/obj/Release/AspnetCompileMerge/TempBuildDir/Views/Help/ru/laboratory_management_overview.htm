Жизненный цикл пробы начинается с её приёма в систему и заканчивается её уничтожением; при этом он включает в себя следующие этапы:

1.Приём в систему. Запись о пробе добавляется в ЭИСНЗ путём либо первоначального создания, либо посредством приёма из другой организации. В момент приёма необходимо ввести основные данные о пробе: лабораторный идентификатор пробы (который вводится вручную или присваивается автоматически, в соответствии со Схемой уникальной нумерации), её состояние в момент приёма, место хранения, и т.д. Помимо этой информации, введите данные о её местонахождении в хранилище ЭИСНЗ, дату приёма в систему, и т.д. Принимаемая в систему проба может быть либо уже связанной со случаем заболевания / сессией, либо ещё не связанной; в первом случае часть информации из записи о случае заболевания / сессии будет автоматически скопирована в некоторые из полей пробы. После этой операции проба получает статус Принята в систему.

Для получения более подробной информации о приёме в систему, прочитайте следующие разделы:

·Приём в систему существующей пробы, связанной со случаем заболевания / сессией (ОФ);

·Регистрация и приём в систему пробы, не связанной со случаем заболевания/сессией (ЛР);

·Приём в систему существующих(-ей) проб(-ы), связанных(-ой) со случаем заболевания / сессией (ЛР);

·Приём в систему группы (ОФ);

·Приём в систему группы (ЛР);

2.Создание аликвот и/или дериватов. При необходимости, пробу со статусом Принята в систему можно использовать для создания нескольких аликвот и/или дериватов. Вместе с тем этот шаг не является обязательным и его можно пропустить. Вновь созданные пробы (аликвоты или дериваты) получают статус Принят(-а) в систему; статус же исходной пробы не меняется.

Для получения более подробной информации, прочитайте следующие разделы:

•Создание аликвоты (ОФ);

•Создание аликвоты (ЛР);

•Создание деривата (ОФ);

•Создание деривата (ЛР).

3.Назначение тестов. После приёма пробы в систему, в её отношении можно планировать проведение тестов. Если проба связана со случаем заболевания или сессией, диагноз такого случая заболевания / сессии определяет список доступных для неё тестов. Если у пробы нет ни связанного случая заболевания, ни сессии, будет доступен полный список тестов. Тесты получают статус Не начат, статус самой пробы - не меняется.

Для получения более подробной информации, прочитайте следующие разделы.

•Назначение тестов во время приёма пробы в систему (ОФ);

•Назначение тестов в форме «Список проб» (ОФ);

•Назначение тестов из формы «Проба» (ОФ);

•Назначение теста (ЛР).

4.Проведение тестов. После назначения теста, можно приступить к его проведению. Для этого измените его статус на статус В процессе т.е., тест начат, но ещё не завершён. Этот шаг можно опустить, если назначение теста осуществлялось тогда, когда его результат уже был известен и его можно было ввести в систему.

Для получения более подробных сведений, прочитайте разделы Начало теста (ОФ) и Начало тестов (ЛР).

5.Ввод результата теста. По завершении теста, введите его результат в систему ЭИСНЗ.

•Если какой-либо результат теста не был утверждён (закрыт), введите его через лабораторный раздел, и установите ему статус Предварительный. В этом случае утверждение теста потребует регистрации в качестве дополнительного шага.

Для получения более подробной информации, прочитайте раздел Ввод результата теста (ЛР).

•Если результат теста уже утверждён (закрыт), введите его в систему либо в помощью двух шагов через форму Лабораторный раздел (как описано выше), либо с помощью одного шага, через форму Проба. После утверждения теста, его статус меняется на Окончательный.

Для получения более подробной информации, прочитайте раздел Ввод результата теста (ЛР).

•Если тест проводили во внешней лаборатории, его результат вводится в качестве результата внешнего теста.

Для получения более подробной информации, прочитайте разделы Указание результата теста, проводившегося во внешней лаборатории (ОФ) и Указание результата теста, проводившегося во внешней лаборатории (ЛР).

6.Утверждение результата теста. После ввода результата теста в систему и если это ещё не было сделано ранее, он проходит утверждение (подтверждение) уполномоченным лицом. После утверждения, тест получает статус Окончательный.

Для получения более подробной информации, прочитайте раздел Утверждение результата теста (ЛР).

7.Изменение (отклонение) результата теста. Если после утверждения результата теста окажется, что он был неправильным, результат теста необходимо изменить. В этом случае тест получает статус Изменён. Этот шаг не является обязательным и, обычно, пропускается. Статус Изменён представляет собой финальное завершение цикла теста; после этого для него возможен путь только либо другого изменения, либо стандартного уничтожения. Вместо повторения этих действий с изменённым тестом, назначьте проведение нового теста.

Для получения более подробной информации, прочитайте разделы Изменение результата теста (ОФ) и Изменение результата теста (ЛР).

8.Уничтожение проб. После завершения и утверждения всех тестов, пробу вначале надо отметить на уничтожение (статус Отмечена для стандартного уничтожения), а затем уничтожить (статус Уничтожена). После уничтожения, запись о пробе хранится в системе и может быть найдена поиском.

Для получения более подробной информации, прочитайте следующие разделы:

•Отметка пробы для уничтожения (ликвидации) (ОФ);

•Отметка пробы для стандартного уничтожения (ЛР);

•Уничтожение пробы (ОФ);

•Уничтожение пробы (ЛР).

Помимо этого, любую пробу со статусом Принята в систему и не имеющую тестов со статусом В процессе, можно передать в другую организацию для проведения тестов, исследований, и т.д. внешней лабораторией. Для получения более подробной информации, прочитайте разделы Передача пробы во внешнюю организацию (ОФ) и Передача пробы во внешнюю организацию (ЛР).

После проведения тестов на пробе, их результаты можно зарегистрировать в системе. Для получения более подробной информации, прочитайте разделы Указание результата теста, проводившегося во внешней лаборатории (ОФ) и Указание результата теста, проводившегося во внешней лаборатории (ЛР).

Переданную во внешнюю организацию пробу можно получить обратно. Для получения более подробной информации, прочитайте разделы Приём в систему / получение передававшихся(-ейся) существующих(-ей) проб(-ы), связанных(-ой) со случаем заболевания / сессией (ЛР) и Обратное получение пробы.

Тесты можно проводить группами, представляющими собой наборы тестов, объединённые для выполнения аналогичного теста. Как правило, это отражает штатив для пробирок, используемый для одновременного проведения теста. Для получения более подробной информации, прочитайте следующие разделы: Работа с группами;, Создание группы в форме «Список тестов»;, Создание группы в форме «Список групп»;, Закрытие группы (ОФ)..
