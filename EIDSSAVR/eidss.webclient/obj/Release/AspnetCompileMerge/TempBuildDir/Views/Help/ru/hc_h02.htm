Рис. 209 Раздел «Сводная информация о случае заболевания»

Поле / переменное значение

Описание

Идентификатор случая заболевания

Поле Идентификатор случая заболевания содержит уникальный номер случая заболевания, который присваивается такому случаю при первом сохранении. Идентификатор случая заболевания генерируется в соответствии со Схемой уникальной нумерации.

Статус случая заболевания

В раскрывающемся списке поля Статус случая заболевания имеются два значения: Закрыт и В работе: Если расследование случая заболевания было завершено или заболевание не подтвердилось, необходимо выбрать опцию Закрыт, чтобы все поля формы, за исключением Статус случая заболевания, стали недоступны для редактирования. Опцию В работе необходимо выбирать тогда, когда расследование случая заболевания ещё не завершено. По умолчанию, статусом случая является значение В работе.

Дата регистрации /
Дата последнего сохранения

Поле Дата регистрации заполняется автоматически при создании записи о случае заболевания в системе. После каждого сохранения случая заболевания, в поле Дата последнего сохранения подставляется текущая дата. Сведения этих двух полей всегда информируют пользователя о дате создания и дате последнего изменения случая заболевания. При создании случая заболевания, значения в этих полях будут одинаковыми. При регистрации нового случая заболевания, значением по умолчанию поля Дата регистрации будет значение текущей даты, а поле Дата последнего сохранения будет пустым. Значение поля Дата регистрации при последующем вводе данных остаётся неизменным. Значение поля Дата последнего сохранения обновляется каждый раз, когда информация о случае заболевания сохраняется.

Пациент

В поле Пациент отображаются фамилия, имя и отчество пациента. Эти данные берутся с вкладки Извещение. Значения в этом поле остаются пустыми до заполнения полей Фамилия, Имя и Отчество в группе Демографическая информация.

Текущий диагноз

Значения в полях Диагноз или Изменённый диагноз появляются после заполнения соответствующего поля на вкладке «Извещение».

Классификация случая заболевания

Поле Классификация случая заболевания берёт данные либо из поля Первоначальная классификация, либо из поля Окончательная классификация, которые находятся на вкладке «Классификация случая заболевания». Значение поля Окончательная классификация заполняется автоматически, в соответствии с определением случая заболевания. При отсутствии окончательной классификации, это поле остаётся заполненным той информацией, которая была внесена в него автоматически при первоначальной классификация случая заболевания.

Зарегистрировавший работник

В поле Работник указывается сотрудник, который ввёл данные по извещению.

Организация

В поле Организация указывается учреждение, в котором числится работник, зарегистрировавший данные по извещению.
Вкладка «Извещение» содержит разделы для ввода общих сведений, демографической и клинической информации о случае заболевания. Эту вкладку надо заполнять на основании стандартной формы экстренного извещения.

Рис. 210 Раздел данных «Бумажная форма»

Поле / переменное значение

Описание

Дата заполнения
бумажной формы

Когда заполняется бумажная форма расследования случая заболевания, происходит и заполнение поля Дата заполнения бумажной формы. По умолчанию это поле остаётся пустым, его значение можно ввести двумя способами: либо набрав значение с клавиатуры в само поле (при этом датой по умолчанию будет текущая дата, а временем - 12:00:00 дня), либо выбрав значение мышкой из календаря (датой по умолчанию будет текущая дата).

Локальный идентификатор

В поле Локальный идентификатор пользователь обязан ввести локальный номер, присвоенный случаю заболевания в бумажной форме экстренного извещения. Это поле содержит текст в свободной форме.

Диагноз

Для занесения информации в поле Диагноз, из раскрывающегося списка необходимо выбрать значение первоначального диагноза на момент уведомления. Это поле выделяется красной рамкой и жёлтым фоном, что означает обязательность его заполнения. Без указания диагноза, поле сохранить невозможно.

Дата диагноза

В поле Дата диагноза указывается значение даты первоначального диагноза. Это поле остаётся недоступным для редактирования и активируется только при выборе диагноза. По умолчанию поле Дата диагноза остаётся пустым, его значение можно ввести двумя способами: либо набрав значение с клавиатуры в само поле (при этом датой по умолчанию будет текущая дата), либо выбрав значение мышкой из календаря (датой по умолчанию будет текущая дата).

Рис. 211 Раздел демографической информации

Поле / переменное значение

Описание

Ф.И.О.: фамилия, имя, отчество.
 

В поле Фамилия пользователь должен в произвольной текстовой форме ввести фамилию пациента или выбрать её из списка, нажав кнопку . После выбора в списке нужного пациента, некоторые из полей такого случая заболевания автоматически заполняются данными, взятыми из формы «Подробные сведения о человеке».
Заполнение поля Фамилия является обязательным. Запись о случае заболевания без указания информации в поле Фамилия сохранить невозможно. Поля Имя и Отчество не являются обязательными для заполнения, но в них в произвольной текстовой форме можно вписать имя и отчество пациента. Находящееся в разделе Краткая информация о случае заболевания поле Пациент будет являться сочетанием значений полей Имя, Фамилия и Отчество.

Возраст / дата рождения

Возраст пациента можно указать двумя способами. Пользователь может указать точное значение в поле Дата рождения, и тогда возраст будет рассчитан автоматически, или просто может ввести значение в поле Возраст. Второй способ можно использовать в тех случаях, когда точная дата рождения пациента неизвестна. Вручную одновременный ввод данных в поля Возраст и Дата рождения - невозможен. Одно из полей всегда остаётся редактируемым. По умолчанию поле Дата рождения остаётся пустым, его значение можно ввести двумя способами: либо набрав значение с клавиатуры в само поле (при этом датой по умолчанию будет текущая дата), либо выбрав значение мышкой из календаря (датой по умолчанию будет текущая дата). Возраст является числовым полем, в которое можно вводить только цифры. Данные о возрасте пациента могут вводиться в поле Возраст при помощи выбора значений из раскрывающегося списка: дни, месяцы, годы. По умолчанию значения в этом поле отсутствуют.

Пол

В поле Пол, значение пола пациента выбирается из раскрывающегося списка: Мужской или Женский.

Тип личного идентификатора

В поле Тип личного идентификатора, из раскрывающегося списка выбирается значение типа личного идентификатора пациента.

Личный идентификатор

В поле Личный идентификатор указывается значение личного идентификатора пациента. Это поле является активным только в случае выбора поля Тип личного идентификатора.

Область / район

Заполнение информации в поля Область и Район, входящие в группу Текущее место жительства, является обязательным. Пока поля Область и Район не заполнены, случай заболевания сохранить невозможно. Значения для заполнения этих полей можно брать из раскрывающихся списков областей и соответствующих районов. Значение поля Район нельзя выбрать, пока не будет указано значение в поле Область.

Населённый пункт

Значение поля Населённый пункт можно выбирать из раскрывающегося списка. При этом значение в поле Населённый пункт выбрать невозможно, пока не будет указано значение в поле Район.

Улица

Поле Улица представляет собой раскрывающийся список. Изначально этот раскрывающийся список пуст, поскольку системе нецелесообразно хранить полный перечень всех улиц всех населённых пунктов страны. Пользователь может ввести название улицы населённого пункта вручную. Заполнение поля Улица возможно только после заполнения поля Населённый пункт.

Дом/корпус/квартира

В полях ввода текста в произвольной форме Дом/корпус/квартира пользователь может указать дом проживания пациента. Заполнение полей Дом/корпус/квартира возможно только после заполнения поля Населённый пункт.

Почтовый индекс

Пользователь может заполнять поле Почтовый индекс аналогично заполнению поля Улица. Заполнение поля Почтовый индекс возможно только после заполнения поля Населённый пункт. Текстовая информация вносится в это поле в произвольной форме.

Номер телефона

Поле для текстовой информации в произвольной форме Номер телефона служит для указания номера телефона пациента.

Гражданство

В поле Гражданство пользователь может выбрать подданство или гражданство пациента. Значение можно выбрать из раскрывающегося списка. Новые значения в такой раскрывающийся список можно добавлять в форме А01. Пользователь может открыть эту форму нажав кнопку , расположенную в поле Гражданство.

Географическая долгота/широта

Пользователь может вводить значения географической долготы и широты места вручную, используя кнопки управления «вверх»/«вниз», или посредством нажатия кнопки Карта (открывается модуль ГИС).

Поле Географическая долгота является числовым. Числа можно вводить в следующем формате: x.xxxxx, и исключительно в диапазоне от -180 до +180.

Поле Географическая широта является числовым. Числа вводятся в следующем формате: x.xxxxx, и исключительно в диапазоне от -85 до +85.

При указании пользователем определённой точки на карте, значения в полях Географическая долгота и Географическая широта заполняются автоматически.

Место работы

Здесь можно указать Место работы пациента. В этом поле указывается место работы, школа, детский сад, учреждение или другое место, которое пациент посещает на регулярной основе, вживую контактируя с большим количеством людей. Поле Место работы является полем для текстовой информации в произвольной форме.

Адрес места работы

Поле Адрес места работы открывает отдельное раскрывающееся окно. Пользователь может указать в нём следующие данные:

–Страна: по умолчанию ставится значение поля той страны, в которой расположен текущий сайт;

–Область: зависит от выбранного значения Страна;

–Район: зависит от выбранной Области, и его невозможно указать, пока не будет выбрана Область;

–Населённый пункт: зависит от выбранного Района; его невозможно указать до выбора значения в поле Район;

–Улица и Почтовый индекс и Дом/корпус/квартира: зависят от выбранного Населённого пункта; значения в этих полях указать невозможно, пока не будет выбрано значение в поле Населённый пункт.

Дата последнего посещения места работы, учёбы, дошкольного учреждения

По умолчанию поле Дата последнего посещения места работы, учёбы, дошкольного учреждения остаётся пустым, его значение можно ввести двумя способами: либо набрав значение с клавиатуры в само поле (при этом датой по умолчанию будет текущая дата), либо выбрав значение мышкой из календаря (датой по умолчанию будет текущая дата).

Поиск дубликатов

Кнопка Поиск дубликатов позволяет проводить поиск среди уже зарегистрированных в системе случаев заболеваний, которые могут оказаться аналогичными создаваемому случаю. Случаи заболеваний сравниваются по следующим параметрам: первоначальный диагноз, окончательный диагноз, локальный идентификатор случая заболевания, фамилия, имя, отчество, возраст пациента, размерность возраста пациента.

Рис. 212 Раздел «Клиническая информация»

Поле / переменное значение

Описание

Дата появления симптомов

В поле Дата появления симптомов пользователь должен указать день первого проявления симптомов у пациента. По умолчанию поле Дата появления симптомов остаётся пустым, его значение можно ввести двумя способами: либо набрав значение с клавиатуры в само поле (при этом датой по умолчанию будет текущая дата), либо выбрав значение мышкой из календаря (датой по умолчанию будет текущая дата).

Состояние пациента на момент извещения

Поле Состояние пациента на момент извещения может иметь статус Скончался или Жив. Статус выбирается из раскрывающегося списка.

Изменённый диагноз (если известен)

Поле Изменённый диагноз (если известен) заполняется в том случае, если в процессе расследования случая заболевания диагноз был изменён. Выбор значения изменённого диагноза осуществляется из раскрывающегося списка. Если изменённый диагноз был указан, необходимо указать причину такого изменения. Причина указывается в отдельном окне, появляющемся сразу после заполнения поля Изменённый диагноз (если известен) (форма H12 ***). Это окно невозможно закрыть без указания причины. После этого, нажатие кнопки в поле Изменённый диагноз (если известен), в отдельном окне (формы H14), вызывает Историю изменений диагноза.

Дата изменения диагноза

Поле Дата изменения диагноза остаётся недоступным для редактирования, пока не будет указано значение в поле Изменённый диагноз. При выборе значения Изменённый диагноз, у пользователя появляется возможность вручную указать дату.По умолчанию поле Дата изменения диагноза остаётся пустым, его значение можно ввести двумя способами: либо набрав значение с клавиатуры в само поле (при этом датой по умолчанию будет текущая дата), либо выбрав значение мышкой из календаря (датой по умолчанию будет текущая дата).

Текущее местонахождение пациента

Поле Текущее местонахождение пациента позволяет указать местонахождение пациента на момент извещения: Дом, Больница, Другое. При выборе местонахождения в больнице, из раскрывающегося списка можно выбрать Название больницы (при выборе иного значения, это поле будет недоступно); если же он/она находится и не дома и не в больнице, в поле Название другого местонахождения можно указать название такого места, используя текстовое поле (при выборе иного значения, это поле будет недоступно).

Дополнительная информация / комментарии, включая перемещение пациента в течение максимального инкубационного периода

В текстовое поле Дополнительная информация / комментарии, включая перемещение пациента в течение максимального инкубационного периода можно добавлять дополнительные важные данные.

Рис. 213 Раздел общей информации и информации извещений

Поле / переменное значение

Описание

Дата извещения

В поле Дата извещения пользователь указывает дату отправки извещения.По умолчанию поле Дата извещения остаётся пустым, его значение можно ввести двумя способами: либо набрав значение с клавиатуры в само поле (при этом датой по умолчанию будет текущая дата), либо выбрав значение мышкой из календаря (датой по умолчанию будет текущая дата).

Отправитель извещения /

получатель извещения

Строки Отправитель извещения / получатель извещения содержат раскрывающиеся списки, в которых можно выбрать как Ф.И.О., так и Учреждение. Помимо этого, учреждение можно найти поиском, нажав кнопку . Новые значения в раскрывающиеся списки можно добавлять нажимая кнопку .
Вкладка «Расследование случая заболевания» содержит несколько вложенных вкладок, на которых отображаются демографические данные, клиническая информация, сбор проб, перечень связанных со случаем заболевания лиц, классификация случая заболевания (клинические признаки), эпидемиологические связи, факторы риска, окончательная классификация случая заболевания и его исход. Вкладка «Расследование случая заболевания» заполняется на основе уже заполненной стандартной бумажной формы «Расследование случая заболевания».

Первоначальная классификация случая заболевания

Поле Первоначальная классификация случая заболевания может заполняться данными из раскрывающегося списка и содержать одно из трёх значений, характеризующих диагноз заболевания: Случай подтвердился, Случай представляется вероятным или Случай представляется сомнительным.

–Опцию Подтвердился следует выбирать, если случай заболевания был подтверждён проведёнными контрольными лабораторными тестами независимо от наличия или отсутствия клинических симптомов и эпидемиологических связей.

–Опция Представляется вероятным указывает на то, что заболевание имеет либо клиническую и/или эпидемиологическую связь, и/или предположительный тест дал положительную реакцию.

–Опцию Представляется сомнительным следует выбирать, если у случая заболевания в наличии только клинические симптомы.

Рис. 216 Рис. 217 Вкладка «Расследование случая заболевания» - вложенная вкладка «Взятие проб»

На вкладке Взятие проб вводится информация о взятых пробах. Поле Взятие проб содержит раскрывающийся список, состоящий из значений: Да, Нет, Неизвестно. По умолчанию значение в этом поле остаётся невыбранным. Если пробы не брались (для поля Взятие проб указано значение Нет), необходимо указать причину ( появится новое текстовое поле Причина). В этом случае функции добавления и удаления проб будут неактивны.

Если факт взятия или невзятия проб неизвестен (для поля Взятие проб указано значение Неизвестно), на этой вкладке не будет никаких дополнительных полей, а функции добавления и удаления проб будут неактивны.

Если проба была принята сторонней лабораторией, информация о таких пробах остаётся доступной только для чтения и нередактируемой.

Если пробы брались (значение Да в поле Взятие проб), пользователь может добавлять комментарии о таких пробах. Функции добавления и удаления записей о пробах становятся активными.

Для очистки содержимого поля Взятие проб, необходимо выбрать пустое поле. Как только будет выбрано пустое поле, во всплывающем окне появится запрос для пользователя на подтверждение удаления или оставления содержимого. При подтверждении пользователем удаления, поле будет очищено.

Для добавления записи о новой пробе, пользователю необходимо указать факт взятия проб (значение Да для поля Взятие проб), и нажать кнопку Нажать для добавления новой строки. В таблице будет создана новая запись.

Рис. 219 Вкладка «Расследование случая заболевания» - вложенная вкладка «Классификация случая заболевания», вид представления данных для ботулизма; для других заболеваний предусмотрен другой вид представления данных

Симптомы пациента указываются на вкладке Классификация случая заболевания. Набор полей на этой вложенной вкладке зависит от предположительного диагноза пациента, указанного на вкладке Извещение. При отсутствия указания диагноза, эта вложенная вкладка остаётся пустой. Вид этой вкладки определяется в конструкторе настраиваемых форм (Система / Конфигурация / Конструктор настраиваемых форм), в Редакторе шаблонов или Клинических симптомах заболевания человека. В случае отсутствия назначенного для определённого диагноза шаблона, применяется единый шаблон. Динамические параметры существующему списку параметров на вложенной вкладке Классификация случая заболевания можно добавлять только для текущего случая заболевания человека. Чтобы добавить параметр, пользователь нажимает кнопку Добавить параметр, а чтобы удалить динамически добавленный параметр - нажимает кнопку Удалить параметр.

Рис. 220 Вкладка «Расследование случая заболевания» - пункт «Эпидемиологические связи и факторы риска». Вид представления данных для ботулизма; для других заболеваний предусмотрен другой вид представления данных

На вложенной вкладке Эпидемиологические связи и факторы риска содержатся дополнительные данные, которые важны для постановки правильного диагноза. Для каждого диагноза отображается соответствующий набор полей. Такие поля относятся к данному определённому диагнозу. При отсутствия указания диагноза, эта вложенная вкладка остаётся пустой. Вид этой вкладки определяется в конструкторе настраиваемых форм (Система / Конфигурация / Конструктор настраиваемых форм), в Редакторе шаблонов раздела Эпидемиологические расследования заболеваний людей. В случае отсутствия назначенного для данного определённого диагноза шаблона, применяется единый шаблон.

Рис. 221 Вкладка «Расследование случая заболевания» - вложенная вкладка «Окончательная классификация и исход случая заболевания»

Вкладка Окончательная классификация и исход случая заболевания заполняется тогда, когда известны окончательная классификация случая заболевания и его исход, и также когда случай заболевания связан со вспышкой заболевания.

 

Поле / переменное значение

Описание

Окончательная классификация случая заболевания

Поле Окончательная классификация случая заболевания может заполняться одним из пяти значений из раскрывающегося списка: Случай подтвердился, Случай представляется вероятным, Более не отслеживается, Не является случаем заболевания или Случай представляется сомнительным.

–Опцию Подтвердился следует выбирать, если случай заболевания был подтверждён проведёнными контрольными лабораторными тестами независимо от наличия или отсутствия клинических симптомов и эпидемиологических связей.

–Опция Представляется вероятным указывает на то, что заболевание имеет либо клиническую и/или эпидемиологическую связь, и/или предположительный тест дал положительную реакцию.

–Опцию Представляется сомнительным следует выбирать, если у случая заболевания в наличии только клинические симптомы.

–Опция Более не отслеживается означает, что расследование случая заболевания прекращено ввиду того, что его отслеживание затруднено или невозможно.

–Опция Не является случаем заболевания означает, что случай заболевания не подтвердился.

Дата окончательной классификации случая заболевания

В этом поле указывается дата окончательной классификации случая заболевания. Такое поле становится доступным для редактирования сразу после внесения значения в поле Окончательная классификация случая заболевания. Это значение можно ввести двумя способами: либо набрав значение с клавиатуры в само поле (при этом датой по умолчанию будет текущая дата), либо выбрав значение мышкой из календаря (датой по умолчанию будет текущая дата).

Окончательный диагноз / дата окончательного диагноза

Поля Окончательный диагноз и Дата окончательного диагноза заполняются на основе данных из вкладки Извещение. Эти поля недоступны для редактирования. Если первоначальный диагноз меняется, то значение для поля Окончательный диагноз берётся из поля Изменённый диагноз (если известен), а значение поля Дата окончательного диагноза берётся из поля Дата изменения диагноза . Если первоначальный диагноз не меняется, то значение для поля Окончательный диагноз берётся из поля Диагноз, а значение поля Дата окончательного диагноза берётся из поля Дата диагноза .

Основание для постановки диагноза

Основание диагноза указывается посредством выбора кнопок с фиксацией: Клинические признаки, Эпидемиологические связи, Лабораторный тест.

Исход заболевания

В поле Исход случая заболевания пользователь может выбрать из раскрывающегося списка одно из трёх значений: либо пациент Выздоровел(-а), Умер(-ла), либо его/её состояние Неизвестно. По умолчанию значение в этом поле не выбрано.

Дата выписки

В поле Дата выписки указывается дата, в которую пациент был выписан из больницы. По умолчанию это поле остаётся пустым, его значение можно ввести двумя способами: либо набрав значение с клавиатуры в само поле (при этом датой по умолчанию будет текущая дата), либо выбрав значение мышкой из календаря (датой по умолчанию будет текущая дата). Поле доступно для редактирования только в случае если значением поля Исход случая заболевания указано значение Выздоровел(-а).

Дата смерти

В поле Дата смерти указывается дата, в которую пациент скончался. По умолчанию это поле остаётся пустым, его значение можно ввести двумя способами: либо набрав значение с клавиатуры в само поле (при этом датой по умолчанию будет текущая дата), либо выбрав значение мышкой из календаря (датой по умолчанию будет текущая дата). Поле доступно для редактирования только в случае если значением поля Исход случая заболевания указано значение Умер(-ла).

Относится ли случай заболевания к вспышке

В поле Относится ли случай заболевания к вспышке указывается относится ли текущий случай заболевания к вспышке заболевания. Из раскрывающегося списка можно выбрать одно из трёх значений: Да, Нет, Неизвестно. По умолчанию значение в этом поле остаётся невыбранным.

Идентификатор вспышки заболевания

Поле Идентификатор вспышки заболевания блокировано по умолчанию. Если случай заболевания относится к вспышке (значением поля «Относится ли случай заболевания к вспышке» является Да), поле Идентификатор вспышки заболевания становится активным и может быть заполнено; в противном случае это поле заблокировано.

 

Значение поля Идентификатор вспышки заболевания можно выбрать из списка записей о вспышках, хранящихся в системе (инструмент «лупа» - формы С10) или создать новую запись о вспышке (инструмент «эллипс» – формы С11).

Комментарии

В поле текста в произвольной форме Комментарии, пользователь может в произвольной форме изложить свои комментарии.

Ф.И.О. эпидемиолога

В поле текста в произвольной форме Ф.И.О. эпидемиолога можно указать имя эпидемиолога, который произвёл окончательную классификацию случая заболевания.

Вкладка «Тесты» предназначена для отслеживания информации о тестах: проводились ли они, просматривать данные о всех тестах, введённые техническими специалистами лаборатории (планируемых, проводимых и завершённых), а также вручную регистрировать результаты тестов непосредственно в записях о случаях заболеваний. Помимо этого, данная вкладка содержит резюмирование и интерпретацию результатов тестов. Для дериватов, столбец Тип пробы содержит тип первоначальной пробы.

Рис. 222 Вкладка «Тесты»

На этой вкладке заполняется информация о проведённых тестах. Поле Тесты проведены содержит раскрывающийся список возможных значений: Да, Нет, и Неизвестно. Если тестирование проводится во внутренней лаборатории, располагающей установленной ЭИСНЗ, это поле заполняется автоматически значением «Да», которое берётся из лабораторного модуля. Если тестирование проводится во внешней лаборатории, пользователь должен ввести это значение вручную.

Если тестирование проводилось в другой лаборатории, информация по таким тестам предназначена только для чтения и является нередактируемой.

Для очистки содержимого поля Тесты проведены, необходимо выбрать пустое поле. Как только будет выбрано пустое поле, во всплывающем окне появится запрос для пользователя на подтверждение удаления или оставления содержимого. При подтверждении пользователем удаления, поле будет очищено.

Для добавления записи о тесте, пользователю необходимо указать факт проведения тестирования (поставить значение Да для поля Тесты проведены), и нажать кнопку Нажать для добавления новой строки. В таблице будет создана новая запись, а поле Локальная проба будет автоматически увеличено.

Примечание: только пользователь с соответствующими правами доступа может добавлять новые тесты.

После сохранения записи её невозможно удалить - кнопка Удалить будет заблокирована.

Диагноз

Название теста и категория теста

Поля Название теста и Категория теста заполняются автоматически информацией из группы полей Тесты по каждому тесту отдельно. Эти поля недоступны для редактирования. Раскрывающееся меню Категория теста включает в себя следующие значения: Подтверждающий, Вероятностный, Процедурный.

Не подтвердился / подтвердился

В столбце Не подтвердился / подтвердился определяется наличие подтверждения либо отсутствие такого подтверждения у диагноза, основанное на результате теста. Значения этого поля выбираются из раскрывающегося списка: Подтвердился, Не подтвердился.

Комментарии

Столбец Комментарии позволяет вводить комментарии к интерпретации результата в поле для текстовой информации в произвольной форме.

Дата интерпретации

Столбец Дата интерпретации хранит дату, в которую диагноз был подтверждён или не подтверждён на основании результатов теста; по умолчанию, такой датой является дата добавления записи в таблицу. Это поле является нередактируемым.

Кем интерпретировано

Столбец Кем интерпретировано заполняется автоматически сразу после указания значения в столбце Не подтвердился / подтвердился. Значением этого столбца является Ф.И.О. пользователя, предоставившего свою интерпретацию в отношении конкретного результата.

Утверждён (Д/Н)

Столбец Утверждён (Д/Н) проверяет наличие официального утверждения интерпретации. Выбор кнопки с фиксацией напротив утверждения доступен только для тех пользователей, у которых имеется право на выполнение функции «Право утверждения интерпретации результатов тестов».

Комментарии

Столбец Комментарии предоставляет возможность оставить текстовый комментарий в произвольной форме. Возможность оставлять комментарии доступна исключительно тем пользователям, у которых имеются права на выполнение функции «Право утверждения интерпретации результатов тестов».

Дата утверждения

Столбец Дата утверждения хранит дату утверждения интерпретации; по умолчанию, этой датой является дата добавления записи в таблицу. Это поле недоступно для редактирования.

Утвердившее лицо

Столбец Утвердившее лицо показывает пользователю того, кто утвердил интерпретацию. Поле является автоматически заполняемым и нередактируемым.
