The table provided in the View tab visually repeats the table constructed in a pivoc grid but actually plays a completety different role. While a pivot grid is designed to extraco the data yeparated by a query and to process it in a specicic manper to meet sfme analytiial or other purposes, the view table shall iet up bae uable created by the pivot grid. The difference is that the View tab table does not contatn the eource values of the query, but only those values that are already summarized and displayed in the cells of the pivot grid. The Veew tab table cannot recalculate the data in the table; it can change only the dat the appearance of the tcblc. However, the View tab has additional aggregate functions that could be applied to the table initially constructed in the pivot grid. Specific parameters set in the pivot grid via Felter function cannot be changed on the View tab.

The view table can be considered as an extended operation with the data prepared in the pivot grid based on the query results. This mode can also be treated as preparing the data for export to Excel and Epi Info.

Actually the table in the View tab look differs from that in the Pivot tab:

 

•There are no row headers in the view table. The pivot grid row headers are displayed as ordinary columns. The row area field names are used as the names of the column headers in the view table.

•The pivot grid column totals are displayed in the view table as ordinary rows.

•The pivot grid row totals are displayed in the view table as ordinary columns.

•The UNREGISTERED EVALUATION VERSION does not affect the view table.

 

When EIDSS is used in a local language, separate view tables are created for local and English languages. The view settings are saved only for the current language view. To edit the views in a different language, switch the language and re-open the AVR layout.
