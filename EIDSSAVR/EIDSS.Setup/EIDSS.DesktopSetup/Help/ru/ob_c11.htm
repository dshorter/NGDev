Рис. 256 Форма «Сведения о вспышке заболевания» 

Окно «Сведения о вспышке заболевания»:

На вкладке «Общая информация» отображается общая информация о вспышке заболевания.

Поля данных

Идентификатор вспышки заболевания

В поле Идентификатор вспышки заболевания отображается уникальный номер вспышки заболевания. Этот номер заполняется при первом сохранении случая заболевания.

Дата начала

В поле Дата начала указывается дата начала вспышки заболевания.

Дата окончания

В поле Дата окончания указывается дата окончания вспышки заболевания.

Диагноз / группа диагнозов

В поле Диагноз / группа диагнозов указывается диагноз или группа диагнозов вспышки заболевания. Из раскрывающегося списка можно выбрать либо специальный диагноз, либо всю группу диагнозов. Это поле является обязательным для заполнения.

Статус вспышки заболевания

В поле Статус вспышки заболевания указывается статус вспышки заболевания. Из раскрывающегося списка можно выбрать следующие статусы: Подтвердилась, Представляется вероятной и Представляется сомнительной.

Основное местоположение вспышки заболевания

В поле Основное местоположение вспышки заболевания указывается географическое местоположение вспышки заболевания. Нажмите кнопку , и на экране раскроется окно формы Географическое местоположение.

Основной случай / сессия заболевания

Указывается основной случай или сессия вспышки заболевания.

Описание

В поле Описание может указываться текстовое описание вспышки заболевания.

 

Раздел Связанные со случаем заболевания / сессией сообщения содержит перечень случаев или сессий заболевания, относящихся к вспышке, и включает в себя следующие параметры:

Поля данных

Идентификатор случая / сессии заболевания

Тут отображается идентификатор случая или сессии заболевания.

Дата случая / сессии заболевания

Тут отображается дата случая заболевания.

В отношении случая заболевания человека: при заполненном поле Дата появления симптомов, значение даты берётся из него;

Если значение в поле Дата появления симптомов не указано, но указано значение в поле «Дата извещения», то отображается значение даты, взятое из поля «Дата извещения».

При отсутствии значений в обоих полях: Дата появления симптомов и Дата извещения, отображается значение, указанное в поле Дата регистрации.

В отношении случая заболевания животного: при заполненном поле Дата расследования, значение даты берётся из него. Если значение в поле Дата расследования не указано, то отображается значение, взятое из поля Дата регистрации данных.

В отношении сессии надзора за переносчиками заболевания: отображается значение из поля Дата начала.

Источник даты случая / сессии заболевания

Отображается название поля того случая / сессии заболевания, значение которого определяет дату случая / сессии заболевания.

Классификация случая заболевания

Отображается классификация случая заболевания.

В отношении случая заболевания человека: если поле Окончательная классификация случая заболевания не заполнено, то отображается значение поля Первоначальная классификация случая заболевания, в ином случае показывается значение поля Окончательная классификация случая заболевания.

В отношении случая заболевания животного: отображается значение поля Классификация случая заболевания.

В отношении сессии надзора за переносчиками заболевания: поле Классификация случая заболевания не заполнено.

Диагноз

Отображается диагноз случая заболевания.

В отношении случая заболевания человека: если поле Изменённый диагноз не заполнено, то отображается значение поля Диагноз, в ином случае отображается значение поля Изменённый диагноз.

В отношении случая заболевания животного: если поле Окончательный диагноз не заполнено, то отображается последнее из значений поля «Предположительный диагноз», в ином случае отображается значение поля Окончательный диагноз.

В отношении сессии надзора за переносчиками заболевания:  отображается значение из поля Заболевание.

Местоположение заболевания

Отображается местоположение случая / сессии заболевания в том виде, как оно определено в поле «Формат географического местоположения».

В отношении случая заболевания человека: указывается местоположение заражения.

В отношении случая заболевания животного: указывается местоположение фермы.

В отношении сессии надзора за переносчиками заболевания: указывается местоположение отлова.

Адрес

Отображается местоположение случая / сессии заболевания в том виде, как оно определено в поле «Формат адреса».

В отношении случая заболевания человека: отображается значение поля «Текущее место жительства».

В отношении случая заболевания животного: отображается значение поля «Адрес фермы».

В отношении сессии надзора за переносчиками заболевания: отображается значение поля «Местоположение отлова».

Пациент / владелец фермы

В этом поле отображается имя пациента или владельца фермы.

В разделе «Связанные со случаем заболевания / сессией сообщения» имеются следующие функциональные кнопки:

Кнопки

Указать в качестве основного/основной

Выберите случай или сессию заболевания, и нажмите эту кнопку для назначения выбранного в качестве основного случая / сессии заболевания для текущей вспышки.

Добавить случай / сессию заболевания -> Добавить случай заболевания животного

Нажмите кнопку Добавить случай заболевания животного , и в новом окне, в режиме выбора, откроется Список случаев заболеваний животных. Вы можете выбрать любой случай заболевания, нажав кнопку Выбрать. Выбранный случай заболевания добавляется в раздел Связанные со случаем заболевания сообщения формы Сведения о вспышке заболевания.

Если выбранный случай заболевания уже был связан с какой-либо вспышкой заболевания, система предложит удалить его связь с текущей вспышкой заболевания и связать его с редактируемой вспышкой заболевания.

При нажатии пользователем кнопки Да, связь случая заболевания со старой вспышкой заболевания удаляется, и назначается с текущей вспышкой. Нажмите кнопку Нет и выделение будет снято.

Если пользователь обладает правом доступа к системной функции Доступ к данным о случаях заболевания животных (хотя бы в режиме чтения), ему будет доступна кнопка Добавить случай заболевания животного.

Примечание: если диагноз или группа диагнозов вспышки заболевания определены, добавить случай заболевания с отличающимся диагнозом не получится.

Добавить случай / сессию заболевания -> Добавить случай заболевания человека

Нажмите кнопку Добавить случай заболевания человека, и в новом окне, в режиме выбора, откроется Список случаев заболеваний людей. Вы можете выбрать любой случай заболевания, нажав кнопку Выбрать. Выбранный случай заболевания добавляется в раздел Связанные со случаем заболевания сообщения формы Сведения о вспышке заболевания.

Если выбранный случай заболевания уже был связан с какой-либо вспышкой заболевания, система предложит удалить его связь с текущей вспышкой заболевания и связать его с редактируемой вспышкой заболевания.

Случай заболевания человека можно добавить к вспышке заболевания только при условии, если значение поля «Относится ли случай заболевания к вспышке» остаётся пустым или указано значение Да. Если значением поля является Нет, то добавление случая заболевания к вспышке невозможно.

Если пользователь обладает правом доступа к системной функции Доступ к данным о случаях заболеваний людей (хотя бы в режиме чтения), ему будет доступна кнопка Добавить случай заболевания человека.

Примечание: если диагноз или группа диагнозов вспышки заболевания определены, добавить случай заболевания с отличающимся диагнозом не получится.

Добавить случай / сессию заболевания -> Добавить сессию надзора за переносчиками заболевания

Нажмите кнопку Добавить сессию надзора за переносчиками заболевания, и в новом окне, в режиме выбора, откроется Список сессий надзора за переносчиками заболеваний. Вы можете выбрать любую сессию, нажав кнопку Выбрать. Выбранная сессия добавляется в раздел Связанные со случаем заболевания сообщения формы Сведения о вспышке заболевания.

Если выбранный случай заболевания уже был связан с какой-либо вспышкой заболевания, система предложит удалить его связь с текущей вспышкой заболевания и связать его с редактируемой вспышкой заболевания.

Примечание: если диагноз или группа диагнозов вспышки заболевания определены, добавить сессию с отличающимся диагнозом не получится.

Просмотреть подробные сведения

Если пользователь выбрал запись о случае / сессии заболевания в разделе Связанные со случаем заболевания сообщения, и дважды нажал на ней мышкой или один раз на кнопке Просмотреть подробные сведения, выбранная запись о случае заболевания откроется в новой форме Сообщение о случае заболевания человека (если это медицинский случай), форме Расследование падежа домашней птицы (если это случай заболевания домашней птицы), форме Расследование заболевания домашнего скота (если это случай заболевания домашнего скота), или форме Сессия надзора за переносчиками заболевания (если это сессия надзора за переносчиками заболевания).

В зависимости от наличия у пользователя прав доступа к системным функциям, такая форма откроется в режимах: для чтения, редактирования и/или удаления.

Удалить

Если в разделе Связанные со случаем заболевания сообщения пользователь выбрал случай заболевания и нажал кнопку Удалить, выбранный случай заболевания будет удалён из вспышки заболевания. Соответствующее поле Идентификатор вспышки заболевания в записи случая заболевания будет очищено.
Вкладка «Примечания» используется для записи любых сведений, относящихся к вспышке заболевания.

Рис. 257  Вкладка «Примечания»

Поля данных

Примечание

В поле Примечание можно добавлять сведения, относящиеся к вспышке заболевания. Для подтверждения написанного примечания, нажмите кнопку ОК.

Для отмены создания примечания или отмены изменения существующего, нажмите кнопку Отменить.

Кнопка OК остаётся блокированной до тех пор, пока какой-либо текст не будет изменён.

Дата

В поле Дата можно указать дату внесения примечания.

Автор

В поле Автор можно указать Ф.И.О. работника, создавшего примечание.
