Фильтр «Существует»

Другим специальным фильтром запроса является фильтр «Существует». Он позволяет осуществлять фильтрацию исходных объектов поиска по факту наличия или отсутствия их специфических дочерних объектов в системе, или дочерних объектов - по наличию некоторых других специфических дочерних объектов аналогичного типа одного и того же исходного объекта поиска. Используемые в фильтре дочерние объекты можно либо использовать, либо не использовать в запросе в качестве дочерних объектов поиска. Более того, фильтр «Существует» не требует наличия полей в панели Список выбранных полей.

Например, необходимо, чтобы какой-либо отчёт включал только те случаи заболеваний людей, по которым брались пробы. Для этого, случай заболевания человека необходимо выбрать в качестве объекта поиска, а критерием его фильтрации будет <Проба случая заболевания человека>.

Постройте фильтр «Существует»

1.Нажмите на названии поля. Откроется раскрывающееся меню.

Рис. 390 Выбор дочерних объектов

2.Из раскрывающегося меню, выберите необходимый дочерний объект; в списке они отмечаются угловыми скобками.

3.После выделения дочернего объекта, к условию фильтрации автоматически добавляется оператор сравнения Существует.

Рис. 391 Пример фильтра «Существует»

Этот фильтр извлекает записи только тех случаев заболеваний людей, у которых имеются зарегистрированные пробы. После этого можно добавлять дополнительные критерии фильтрации как внутри группы фильтра «Существует», так и на одном уровне с ним.

Создание фильтра «Не существует»

Для создания фильтра, извлекающего только те объекты поиска, у которых нет указанных дочерних объектов, фильтр «Существует» должен использоваться вместе с объединяющим оператором НЕ И.

1.Выполните создание фильтра «Существует» как описано выше.

Рис. 392 Фильтр «Существует»

2.Нажмите мышкой на объединяющий оператор И, и в раскрывающемся меню выберите НЕ И.

Рис. 393 Пример фильтра «Не существует»

Этот фильтр извлекает записи только тех случаев заболеваний людей, у которых нет зарегистрированных проб.
