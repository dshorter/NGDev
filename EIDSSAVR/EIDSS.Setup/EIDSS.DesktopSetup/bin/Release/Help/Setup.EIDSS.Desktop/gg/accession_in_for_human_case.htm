დაშვება L05 ფორმაში ადამიანის საქმისთვის

ფორმა დაშვება L05 დამოკიდებულია საქმის ტიპზე ან ამ ნიმუშთან დაკავშირებულ ნიმუშზე. რეალურად, სექციები ნიმუშის დეტალები და დამატებითი ტესტის მოთხოვნა და ნიმუშის შენიშვნები ერთი და იგივეა და აღწერილია დაშვება L05 სექციაში. ერთმანეთისგან განსხვავებულია მხოლოდ ზოგადი ინფორმაციის და ნიმუშის მიღების.რეგისტრაციის სექციები. ადამიანის საქმესთან დაკავშირებული ნიმუშის შესახებ მათი შეხედულება ქვემოთაა მოცემული.

ნახ. 287 დაშვება

ნახ. 288 ზოგადი სესია

ზოგადი ინფორმაციის ფანჯრის ველები ხელმისაწვდომი არ არის, გარდა შემთხვევის ID ველისა, სადაც შეიძლება შეირჩეს სხვა შემთხვევა, რათა ჩამოიტვირთოს მთელი ფორმა.

ხელმისაწვდომობა ველებზე ნიმუშების მიღება/რეგისტრაცია დამოკიდებულია ნიმუშის სტატუსზე. ახალშექმნილი ნიმუშისთვის ხელმისაწვდომია ყველა ველი, გარდატესტებისა მაგრამ მანამ, სანამ ფორმა არის შენახული შენახვის შემდეგ, ველები გადადის მხოლოდ კითხვად /read-only/ რეჟიმში.

არსებული ნიმუშისთვის, ვიდრე მისი მისაწვდომობა ძალაშია, შეიძლება მოხდეს შემდეგი ველების რედაქტირება: ლაბორატორიული ნიმუშის ID, დაშვების თარიღი, მიღებული ნიმუშის მდგომარეობა, კომენტარი, შენახვის ადგილი, და ფუნქციური არეალი. ველები ხელმისაწვდომია ფორმის პირველ შენახვამდე, და შემდეგ ისინიც გადადის მხოლოდ კითხვად რეჟიმში.

ნიმუშების ყველა ველი, რომლებსაც აქვთ დაშვების (Accessioned In) არის მხოლოდ კითხვადი.

 

ზოგადი ინფორმაციის ველები

შემთხვევის ID

საქმის უნიკალური ID. სხვა საქმის გახსნისთვის:

1.დააწკაპუნეთ ძებნის ღილაკზე (). აქ ადამიანის შემთხვევის ანაგარიშის ფორმა H02 გაიხსნება.

2.ან შეარჩიე ბადეში საჭირო საქმე და დააწკაპუნე ღილაკზე შერჩევა ან ორჯერ დააწკაპუნე მას.

საქმის შერჩევის შემდეგ ზებნის ფორმა იხურება. შემთხვევის დეტალებმა ავტომატურად უნდა შეავსონ ზოგადი ინფორმაციის ველების ფანჯარა.

საქმის დეტალების ნახვის მიზნით დააწკაპუნეთ ღილაკზე დეტალები () ღილაკი. აქ ადამიანის შემთხვევის ანაგარიშის ფორმა H02 ფორმა გაიხსნება.

მუდმივი საცხოვრებელი ადგილი

მისამართი სადაც პაციენტი ცხოვრობს ამჟამად

ნებისმიერი ანტიბიოტიკის მიღება ნიმუშების აღებამდე

მონაცემები რაიმე ანტიბიოტიკის მიღების შესახებ ნიმუშების აღებამდე შემთხვევის ველიდან.

ნიმუშის მიღების/რეგისტრაციის ველები

ნიმუშის ტიპი

შეარჩიეთ ტიპი ახალი ნიმუშისთვის. უკვე არსებული ნიმუშის ტიპის შეცვლა არ შეიძლება.

ტესტები

ეკრანზე ჩნდება ნიმუშზე დანიშნული ტესტების რაოდენობა.

ნიმუშის ადგილობრივი ID

შეიყვანეთ ახალშექმნილი ნიმუშის ადგილობრივი ID. არსებულკი ნიმუშისთვის ადრე განსაზღვრული ნიმუშის ადგილობრივი ID გამოჩნდება ეკრანზე და მისი შეცვლა შეუძლებელია.

ლაბორატორიული ნიმუშის საიდენტიფიკაციო კოდი

ნიმუშის ლაბორატორიული ID ავტომატურად შეიქმნება ახალი ნიმუშისთვის და შეიძლება ხელით შევცვალოთ ფორმის პირველ დამახსოვრებამდე. არსებული ნიმუშისთვის, ველი მხოლოდ კითხვადია.

შეგროვების თარიღი

შეიყვანეთ ნიმუშის აღების თარიღი. ხელმისაწვდომია მხოლოდ ახალშექმნილი ნიმუშებისთვის; მანამდე დაშვებულთათვის ეს ველი არის მხოლოდ კითხვადი.

მიღების თარიღი

მიუთითეთ ნიმუშის დაშვების თარიღი. მიმდინარე თარიღი გამოიყენება ვარიანტისთვის და მისი შეცვლა შეიძლება. დაშვებული ნიმუშებისთვის ველი არის მხოლოდ კითხვადი.

ნიმუშის მდგომარეობა მიღებისას

აღწერეთ ნიმუშის მდგომარეობა დაშვების მომენტისთვის. თუკი მდგომარეობა კარგი მდგომარეობის სტატუსისაგან განსხვავდება, საჭიროა ლპმენტარი ჩაიწეროს კომენტარის სვეტში.

ნიმუშის შენახვის შემდეგ ველი ხელმიუწვდომელი გახდება.

კომენტარი

ეკრანზე გამოჩნდება კომენტარები. ნიმუშისთვის, რომელიც უარყოფილი იქნა ან ცუდ მდოგმარეობაში მყოფი ნიმუშის მიღების შემთხვევაში ეს ველი აუცილებელია და მოითხოვს განმარტებას.

შენახვის ადგილი

შეარჩიე შენახვის ადგილი ახალ დაშვებული ნიმუშისთვის. ველი ხელმსიაწვდომია ფორმის პირველ შენახვამდე, ნიმუშის დაშვების შემდეგ.

ფუნქცონალური არე

შეარჩიე ორგანიზაცისი დეპარტამენტი, სადაც შენახულია ნიმუში და მოხდა მისი ანალიზი. გამოიყენება მხოლოდ იმ ნიმუშებისთვის, რომლებიც კარგი ან ცუდი მდგომარეობის სტატუსითაა მიღებული.

ღილაკები

ძებნა

პოულობს და ამატებს ნიმუშების ნუსხას მათი სრული ადგილობრივი ან ველის ID-ის მეშვეობით, რომელიც შეყვანილია საძიებო სისტემის ძებნა ნიმუშის ადგილობრივი/ველის ID-ის მიხედვით ველში.

1.შეიყვანეთ ნიმუშის ადგილობრივი ან საველე ID ძებნა ნიმუშის ადგილობრივი/საველე ID-ის მიხხედვით ველში.

2.დააწკაპუნეთ ძებნის ღილაკზე. ჩამონათვალს დაემატება ყველა ნიმუში, რომელიც შეყვანილი იყო ადგილობრივი/საველე ნიმუშის ID-ის ფარგლებში,

შტრიხკოდების ამობეჭდვა

დაბეჭდავს შტრიხკოდის ეტიკეტებს დაშვებული ნიმუშებისთვის. შეიძლება დაიბეჭდოს მარკირებები მხოლოდ იმ ნიმუშებისთვის, რომლებიც კარგი ან ცუდი მდგომარეობის სტატუსითაა მიღებული.

1.დამახსოვრება ფორმის.

2.შეარჩიეთ უკვე დაშვებული და არა უარყოფილი ნიმუში.

3.დააწკაპუნეთ ბეჭდვის შტრიხკოდების ღილაკზე. გაიხსნება შტრიხკოდის ეტიკეტების ბეჭდვის ფორმა შტრიხკოდის წინასწარი გამოსახულებით.

4.დააწკაპუნეთ ბეჭდვის ღილაკზე ().

დაშვება

შერჩეული ნიმუშების დაშვება. დაშვება მოხდება მხოლოდ იმ ნიმუშების, რომლებიც ჯერ დაშვებული არ არის.

1.შეარჩიეთ ბადეში ნიმუშები, რომლებიც დაშვებული არ არის.

2.დააწკაპუნეთ დაშვების ღილაკზე. შერჩეული ჩანაწერი რედაქტირებადი გახდება.

ახალი

ქმნის ახალ ნიმუშს. დეტალებისთვის, წაიკითხე ნიმუშის დაშვება, დაკავშირებული ან დაუკავშირებელი საქმის/სესიის (IF) სექციასთან.

წაშლა

წაშლის შერჩეულ ნიმუშებს. შეიძლება წაიშალოს მხოლოდ ის ჩანაწერები, რომლებიც დაემატა მიმდინარე დაშვების ფორმის მეშვეობით და არ იყო დამახსოვრებული. ნიმუშის წასაშლელად, შეარჩიეთ იგი და დააწკაპუნეთ წაშლის ღილაკზე.

შეარჩიეთ ტესტი

განსაზღვრავს ტესტს შერჩეული ნიმუშისთვის. დეტალებისთვის, წაიკითხეთ ტესტების განსაზღვრა ნიმუშის დაშვების პროცესში (IF) შესაბამის განყოფილებაში.

1.დააწკაპუნეთ ნიმუშზე ჩამონათვალში.

2.დააწკაპუნეთ ტესტის შერჩევის ღილაკზე. ფორმა შეარჩიე ტესტი L20 გაიხსნება.
