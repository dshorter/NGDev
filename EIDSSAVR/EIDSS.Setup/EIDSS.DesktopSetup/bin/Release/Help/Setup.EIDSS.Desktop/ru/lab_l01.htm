Samples List L01

Форма L01 Список проб создаётся для отображения списка всех зарегистрированных проб, удовлетворяющих критериям поиска. Она состоит из 2-х частей:

•Панель поиска;

•Список проб.

Инструмент «Панель поиска» создан для осуществления самых сложных типов поиска по информации о пробе. В качестве полей поиска, в панели имеются основные и наиболее часто используемые параметры. Остальные параметры можно указывать в полях поиска с текстом в свободной форме, расположенных в нижней части панели.

«Список проб» представляет собой таблицу, каждая из строк которой является записью о пробе, с идентификатором пробы и некоторым количеством параметров пробы, отображаемых в столбцах таблицы. Если некоторые из столбцов не нужны или не актуальны, их можно скрыть, используя инструмент «Селектор столбцов». Больше информации можно найти в разделах Список элементов, , Селектор столбцов, , Списки поиска, а также Настройка списочных и поисковых форм.

Рис. 335 Окно «Список проб»

Поля критериев поиска

Идентификатор случая заболевания / идентификатор сессии

Используется для указания случая заболевания или сессии, связанной с необходимой пробой.

Локальный / полевой идентификатор пробы

Используется для указания локального или полевого идентификатора необходимой пробы.

Диагноз

Используется для указания диагноза случая или сессии заболевания, связанного с соответствующей пробой, выбираемого из раскрывающегося списка. Используются следующие диагнозы:

•Случай заболевания человека: текущий диагноз.

•Случай заболевания животного: окончательный или один из предположительных диагнозов.

•Сессия активного надзора или надзора за переносчиками заболеваний: любой диагноз, связанный с сессией.

Тип пробы

Используется для указания типа соответствующей пробы из раскрывающегося списка.

Лабораторный идентификатор пробы

Используется для указания лабораторного идентификатора соответствующей пробы.

Пациент

Указывается полное либо краткое имя пациента или владельца фермы, относящегося к случаю заболевания, связанному с необходимой пробой.

Владелец фермы

Указывается полное либо краткое имя владельца фермы, относящегося к случаю заболевания, связанному с необходимой пробой.

Дата взятия: с/по

Указывается временной интервал, в течение которого была взята соответствующая проба.

Дата приёма в систему: с/по

Указывается временной интервал, в течение которого соответствующая проба была принята в систему.

Обрабатывающий отдел

Указывается обрабатывающий отдел (подразделение текущей организации), в котором хранится необходимая проба.

Статус

Используется для выбора статуса необходимой пробы. Для получения более подробной информации, прочитайте раздел Статусы пробы.

Статус случая заболевания

Используется для указания статуса случая заболевания, относящегося к необходимой пробе.

Дополнительные поля поиска

Дополнительные поля используются для настраиваемого поиска, когда не хватает присутствующих в панели поиска полей.

В левом поле выберите критерий поиска, а в правом поле укажите для него значение.

 

Столбцы списка результатов

Лабораторный идентификатор пробы

Основная идентификация пробы.

Локальный / полевой идентификатор пробы

Отображается локальный или полевой идентификатор пробы.

Дата приёма в систему

Отображается дата, когда проба(-ы) была/были принята(-ы) в систему.

Тип пробы

Тип пробы.

Дата взятия

Указывается дата взятия пробы.

Обрабатывающий отдел

Указывается обрабатывающий отдел, т.е., отдел организации, которая работает с пробой.

Идентификатор случая заболевания / идентификатор сессии

Отображается идентификатор либо случая заболевания, либо сессии, связанной с пробой.

Диагноз

Отображается диагноз случая заболевания, относящегося к пробе. Если проба связана с сессией, это поле будет пустым.

Пациент / владелец фермы

Пробы случая заболевания человека; отображается фамилия и имя пациента.

Пробы случая заболевания животного; отображается фамилия и имя владельца фермы.

Идентификатор животного / биологического вида

Для пробы, связанной со случаем заболевания домашней птицы или сессией, отображается идентификатор биологического вида.

Для пробы, связанной со случаем заболевания домашнего скота или сессией, отображается идентификатор животного.

Для пробы, связанной со случаем заболевания человека, поле остаётся пустым, поскольку его невозможно применить к такой пробе.

Количество тестов

Отображается общее количество тестов, назначенных для пробы.

Кнопки

Выбор теста

Служит для назначения тестов для выбранных проб:

1.Найдите и выберите пробу(-ы), для которой(-ых) планируется назначить проведение тестаов.

2.Нажмите кнопку Выбрать тест. Откроется форма Выбор теста. Для получения более подробной информации о назначении пробам тестов, прочитайте раздел Назначение тестов в форме «Список проб» (ОФ).

Настоящая кнопка является активной исключительно для проб со статусом Принята в систему.

Отметить для уничтожения

Отмечает выбранную пробу для уничтожения (ликвидации):

1.Укажите пробу(-ы), которую(-ые) планируется удалить.

2.Нажмите кнопку Отметить для уничтожения. Откроется форма Уничтожение пробы. Для получения более подробной информации об отметке проб на уничтожение и последующую ликвидацию, прочитайте разделы Отметка пробы для уничтожения (ликвидации) (ОФ) и Уничтожение пробы (ОФ).

Настоящая кнопка является активной исключительно для проб со статусом Принята в систему.

Передача во внешнюю организацию

Служит для управления передачей выбранных проб во внешнюю организацию:

1.Укажите пробу(-ы), которую(-ые) планируется передать во внешнюю организацию.

2.Нажмите кнопку Передать во внешнюю организацию. Откроется форма Передача пробы. Для получения более подробной информации о передаче проб во внешнюю организацию, прочитайте раздел Передача пробы во внешнюю организацию (ОФ).

Настоящая кнопка является активной исключительно для проб со статусом Принята в систему.

Аликвоты/дериваты

Открывает формы L06 Аликвоты/дериваты для создания записей о либо аликвотах, либо дериватах выбранных проб:

1.Укажите исходную (-ые) пробу(-ы) для создания аликвоты/деривата.

2.Нажмите кнопку Аликвоты/дериваты. Откроется форма Аликвоты/дериваты. Для получения более подробной информации о передаче проб во внешнюю организацию, прочитайте разделы Создание аликвоты (ОФ) и Создание деривата (ОФ).

Настоящая кнопка является активной исключительно для проб со статусом Принята в систему .

Приём в систему

Используется для приёма в систему новых проб.

Нажмите кнопку Приём в систему и, из раскрывающегося меню, выберите либо присоединяемый к пробе объект, либо нажмите Приём в систему группы проб, если в систему будет принята группа проб, объединённых одинаковым локальным / полевым идентификатором. В зависимости от выбранной связи, могут открыться разные формы.

Для получения более подробной информации, прочитайте следующие разделы:

•Приём в систему пробы, связанной или нет со случаем заболевания/сессией (ОФ);

•Приём в систему группы проб (ОФ).

Редактирование

Открывает форму Проба в режиме редактирования выбранной пробы.

1.Укажите пробу для редактирования.

2.Нажмите кнопку Редактировать. Откроется форма Проба. Она создана для редактирования выбранной пробы или назначения для неё тестов. Для получения более подробной информации о назначении тестов, прочитайте раздел Назначение тестов из формы «Проба» (ОФ). Для получения более подробной информации об изменении сведений о пробе, прочитайте раздел Редактирование сведений о пробе (ОФ).

Удаление пробы

Служит для удаления выбранной пробы:

1.Выберите пробу, подлежащую удалению.

2.Нажмите кнопку Удалить пробу. Откроется окно подтверждения на совершение действия.

3.Нажмите кнопку ОК для подтверждения удаления. Для получения более подробной информации об удалении пробы, прочитайте для раздел Удаление пробы (ОФ).
