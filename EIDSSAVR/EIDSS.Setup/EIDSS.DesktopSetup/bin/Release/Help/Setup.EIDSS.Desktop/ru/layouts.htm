Макеты

Макет представляет собой конфигурацию сводной таблицы, связанную с лежащим в её основе аналитическим запросом. В макете указывается размерное представление и схема измерений, используемая для построения в памяти куба OLAP (онлайновой аналитической обработки данных). Помимо этого, макет содержит в себе фильтр тонкой настройки и условия сортировки, применяемые для дополнительного ограничения отображения среза набора данных, а также статистические функции, используемые для обработки отдельных полей с целью их отображения или экспорта.

Перед началом создания макета точно продумайте вид представления данных, в котором должна быть представлена необходимая информация. Извлечённые с помощью запроса данные помещаются в сводную таблицу и обрабатываются соответствующим образом для получения необходимой информации.

В использованном для запроса макете (данные о географическом местоположении определённой болезни) необходимыми для извлечения сведениями является количество случаев заболеваний людей; поэтому поле «Случай заболевания человека - идентификатор случая заболевания» необходимо переместить мышкой на область данных сводной таблицы.

Область размещения географической информации и даты зависит от желаемого результата. Если необходимо построение диаграммы, информацию, которая будет в ней отображаться на оси Х, необходимо перетащить мышкой в область строк. Если требуется географическая карта, необходима информация об административной единице, которую также следует перетащить мышкой в область строк.

Одинаковые поля, используемые разными способами, приводят к созданию разных сводных таблиц.

Пример № 1

Случаи заболеваний людей, сортированные в хронологическом порядке по годам и месяцам, а также географически - по областям и районам.

После помещения данных в сводную таблицу, можно приступить к их обработке. Например, можно узнать количество случаев заболеваний по месяцам или годам. Это зависит от параметра Группировка по дате, применяемого к данным даты. Также возможно узнать информацию как об абсолютном количестве случаев заболеваний людей, зарегистрированных по какому-либо заболеванию, так и относительное значение о населении по стране / области / району. К данным можно применять функции агрегирования.

Пример № 4

Количество случаев заболеваний людей, отфильтрованное по районному населению, извлекается при помощи функции агрегирования Количество на 10.000, и затем округляется до шестого знака. Функция Точность используется для варьирования количества десятичных знаков у значений, если это необходимо.  Дополнительные функции агрегирования выполняются на вкладке Вид представления данных.
