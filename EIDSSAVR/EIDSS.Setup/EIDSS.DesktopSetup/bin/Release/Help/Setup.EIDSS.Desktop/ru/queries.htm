Запросы

Ещё до начала создания запроса или макета, тщательно обдумайте какого типа отчёты или анализ данных требуется. Это наиболее важный и сложный шаг процесса. После принятия решения в отношении концепции, подумайте, какой тип объектов ЭИСНЗ и его полей содержит необходимые данные. Например, если требуются статистические данные о географическом местоположении определённого заболевания, необходимо использовать данные о случаях заболевания людей:

 

■Случай заболевания человека - идентификатор случая заболевания (для идентификации и расчёта количества случаев заболеваний);

■Случай заболевания человека - текущее место жительства пациента - область;

■Случай заболевания человека - текущее место жительства пациента - район.

 

Также следует принять во внимание переменные значения, которые могут использоваться для установки параметров через фильтр.

Например, для выбора определённого заболевания.

 

■Случай заболевания человека - Окончательный диагноз.

 

Если требуется включить какой-либо период, добавьте:

 

■Случай заболевания человека - дата диагноза или случай заболевания человека - дата появления симптомов

 

После этого, когда основные данные будут определены, подумайте, какого рода дополнительная информация может потребоваться. Например, описательный анализ по людям: возрастная группа, пол, исход случая заболевания, и т.д. Добавьте в список:

 

■Случай заболевания человека - возраст;

■Случай заболевания человека - пол;

■Случай заболевания человека - исход случая заболевания.

 

Теперь можно создавать запрос: объект ЭИСНЗ можно добавить в качестве объекта поиска (корневой и/или дочерний). Нужно найти поля из панели Список доступных полей и добавить их в панель Список выбранных полей.

Когда все необходимые данные будут найдены и отделены от остальных данных, отфильтруйте ненужные данные. Например, если требуются статистические данные по определённому заболеванию, остальные диагнозы следует скрыть. Для этого, используйте фильтры. Например, чтобы увидеть только случаи заболевания кожной формой сибирской язвы, установите следующий фильтр:

 

Значение [Случай заболевания человека - диагноз] РАВНО сибирская язва - кожная форма

 

После этого, все необходимые данные определены и могут использоваться в макете.

Если в Конструкторе запросов используется фильтр, создание макетов ограничивается такими параметрами.
