Comparison operators

The Comparison Operator is a part of the filter condition used to select the applied logical operator.

Fig. 519 Comparison operator

Click the Comparison Operator in order to invoke the drop-down menu that allows selection of another logical operator.

 

Fig. 520 Comparison operator drop-down menu

The following table lists the comparison operators available for filter conditions.

Comparison Operator

Description

Application, Exceptions, Examples

Equals

An operator, which retrieves the values equal to the specified value.

Applicable for all types of AVR fields.

Exception:
The Equals operator cannot work for Date and Time fields (Entered Date, Date Last Saved, etc.), meanwhile the Date fields will be processed correctly.

For the Date and Time fields, replace the Equals operator with a set of 2 conditions:

And

[AVR date field] Is greater than or equal to [Specified Date]

[AVR date field] Is less than [Next date after Specified Date]

Does not equal

An operator which retrieves the values that are not equal to the specified value.

Applicable for all types of AVR fields.

Exception:
The Does Not Equal operator cannot work for Date and Time fields (Entered Date, Date Last Saved, etc.), meanwhile the Date fields will be processed correctly.

For the Date and Time fields, replace the Does Not Equal operator with a set of 2 conditions:

And

[AVR date field] Is greater than or equal to [Next date after Specified Date]

[AVR date field] Is less than [Specified Date]

Is greater than

An operator which retrieves the values greater than the specified value (for text fields, in alphabetical order).

Applicable for all types of AVR fields, except text fields connected to EIDSS references (e.g. Diagnosis, Sample Type, Test Name, etc.) or lists (Organizations, EIDSS Sites).

Is greater than or equal to

An operator which retrieves the values greater than or equal to the specified value (for text fields, in alphabetical order).

Applicable for all types of AVR fields, except text fields connected to EIDSS reference values.

Contains

An operator, which retrieves the values containing the specified text.

Applicable for text AVR fields only, except text fields connected to EIDSS references or lists.

Does not contain

An operator, which retrieves the values that do not contain the specified text.

Applicable for text AVR fields only, except text fields connected to EIDSS references or lists.

Begins with

An operator, which retrieves the values beginning with the specified text.

Applicable for text AVR fields only, except text fields connected to EIDSS references or lists.

Ends with

An operator, which retrieves the values ending with the specified text.

Applicable for text AVR fields only, except text fields connected to EIDSS references or lists.

Is like

An operator, which retrieves the values similar to the specified text. The text can include character %, which is defined as any string value.

For example, condition [AVR field] Is like ‘AB%D’ retrieves such values as ‘AB[Any string value]D’ (‘ABXXXXD’, ‘AB12345…D’, etc.)

Applicable for text AVR fields only, except text fields connected to EIDSS references or lists.

Is not like

An operator, which retrieves the values that are not similar to the specified text. The text can include character %, which is defined as any string value.

For example, condition [AVR field] Is not like ‘AB%D’ retrieves any values, except such values as ‘AB[Any string value]D’ (‘ABXXXXD’, ‘AB12345…D’, etc.)

Applicable for text AVR fields only, except text fields connected to EIDSS references or lists.

Is blank

An operator, which retrieves the value Null or empty string.

Applicable for all types of AVR fields.

Is not blank

An operator, which retrieves all values, except Null and empty string.

Applicable for all types of AVR fields.

Is today

An operator, which retrieves the date values equal to today.

The result of this filter depends on the day when the AVR query and/or layout is executed or opened correspondingly.

Applicable for date AVR fields only.

Is yesterday

An operator, which retrieves the date values equal to yesterday.

The result of this filter depends on the day when the AVR query and/or layout is executed or opened correspondingly.

Applicable for date AVR fields only.

Is earlier this week

An operator, which retrieves the date values that meet the following criteria:

First day of this week ≤ Date < Yesterday

The result of this filter depends on the day when the AVR query and/or layout is executed or opened correspondingly.

Applicable for date AVR fields only.

Is last week

An operator, which retrieves the date values that meet the following criteria:

First day of the latest past week ≤ Date < First day of this week

The result of this filter depends on the day when the AVR query and/or layout is executed or opened correspondingly.

Applicable for date AVR fields only.

Is earlier this month

An operator, which retrieves the date values that meet the following criteria:

First day of this month ≤ Date < First day of the latest past week

The result of this filter depends on the day when the AVR query and/or layout is executed or opened correspondingly.

Applicable for date AVR fields only.

Is earlier this year

An operator, which retrieves the date values that meet the following criteria:

First day of this year ≤ Date < First day of this month

The result of this filter depends on the day when the AVR query and/or layout is executed or opened correspondingly.

Applicable for date AVR fields only.

Is prior this year

An operator, which retrieves the date values that meet the following criteria:

Date < First day of this year

The result of this filter depends on the day when the AVR query and/or layout is executed or opened correspondingly.

Applicable for date AVR fields only.

 

Due to specific usage of dates with time, the following filter is correct for retrieving all records within a certain period:

 

And

[Date AVR field] Is greater than or equal to [Start Date]

[Date AVR field] Is less than [Next day after End Date]

 

Please note that the day after the period end date has to be used in the filters like this.
