Layouts

Layout is a configuration of the pivot grid linked to the underlying analytical query. The layout specifies the dimensional representation and measuring scheme used for in-memory OLAP-cube construction. The layout also contains refinement filter and sort conditions for further restricting data set slice as well as statistical functions used to process separate fields in order to display or export it.

Before a layout creation is started, consider precisely the view in which the desired information shall be displayed. After the data is extracted by a query, it is time to place it to a pivot grid, and process in a correct way to receive the required information.

In the example used for query (data about geographical location of some disease), the data we need to extract is the number of human cases, so the Human Case - Case ID should be drag-and-dropped to the data area of a pivot grid.

The area where the geographical and date information needs to be placed depends on the desired result. If a chart is required, the information to be displayed in a chart in the X-axis has to be dragged onto the row area. If a map is required, the administrative unit is mandatory and has to be dragged onto the row area as well.

The same fields used in different ways result in different pivot tables.

When the data is placed to the pivot grid, it is time to process it. For example, the number of cases for months or years can be received. It depends on the Group Date parameter of the Date data. It is also possible to receive both an absolute number of the human cases entered for a disease and a relative value by country/region/rayon population. It can be set up by aggregate functions.

Example 4

The number of human cases by the rayon population is received with the Rate per 10000 aggregate function and then rounded up to the sixth digit. The Precision function is used to change number of decimals of the values if needed.  Additional aggregate functions are performed on the View tab.
