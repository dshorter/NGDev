Human Aggregate Case Details H06

Human Aggregate Case Details (H06) form is used during the creation of the Human Aggregate Case.

Fig. 235 The Human Aggregate Case Details

Field/Variable

Description

Notification Sent by

Institution

The Institution field specifies the organization which sent the notification with an aggregate data.

Officer

The Officer field specifies the employee, who sent the notification with an aggregate data. The Officer field can be filled only if the Institution field is specified.

Date

The Date field specifies the date when the notification with an aggregate data was sent.

 

Notification Received by

Institution

The Institution field specifies the organization which received the notification with aggregate data.

Officer

The Officer field specifies the employee who received the notification with aggregate data. The Officer field can be filled only if the Institution field is specified.

Date

The Date field specifies the date when the notification with aggregate data was received. The Date must be greater than or equal to the Notification Sent by: Date.

 

Entered by

Institution

The Institution field specifies the organization which entered the notification with an aggregate data. It is a non-editable field, which is filled in automatically.

Officer

The Officer field specifies the employee who entered the notification with aggregate data. It is a non-editable field, which is filled in automatically.

Date

The Date field specifies the date and time when the notification with aggregate data was entered in the system. It is a non-editable field, which is filled in automatically.

General Info

Case ID

The Case ID field contains the unique number of the case and will be populated when the case is saved for the first time. The Case ID is generated according to the Unique Numbering Schema after the form is saved for the first time.

Year

The Year field specifies the exact year in which the aggregate data was collected.

Month

The Month field specifies the exact month on which the aggregate data was collected. Visible according to Aggregate settings.

Country

The Country field specifies the country where the aggregate data was collected.  It is a non-editable field.

Region

The Region field specifies the exact region where the aggregate data was collected. Visible according to Aggregate settings. If visible, the field is populated with the current organization region by default and can be manually changed.

Rayon

The Rayon field specifies the exact rayon where the aggregate data was collected. Visible according to Aggregate settings. If visible, the field is populated with the current organization rayon by default and can be manually changed.

Settlement

The Settlement field specifies the exact settlement where the aggregate data was collected. Visible according to Aggregate settings.

 

Diagnosis List

The Diagnosis List section contains a grid. The columns and rows in this grid are country specific and are defined in the Flexible forms designer (Template - columns) and in Human Aggregate Case Matrix Editor (Matrix version - rows).

The template for a Human Aggregate Case is selected from all templates for Human Aggregate Case, where the Determinant is the Country.

The matrix for a Human Aggregate Case is by default the last activated matrix, but it can be changed to another matrix in the list. In this list only activated matrixes are available.

Matrix Version

The field is used to select Matrix Version for Human Aggregate Case. The version is selected from the drop-down list.

Template

The field is used to select Template for Human Aggregate Case. The template is selected from the drop-down list.

Note: The display of Country/Region/Rayon/Settlement and Year/Quarter/Month/Week fields depends on aggregate settings.
