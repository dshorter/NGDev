Case Summary section, which remains displayed when moving through the tabs below. It contains the basic information about the human case.

Fig. 209 Case Summary section

Field/Variable

Description

Case ID

The field Case ID contains the unique number of the case and will be populated when the case is saved for the first time. The Case ID is generated according to the Unique Numbering Schema.

Case Status

In the field Case Status, two values are available in a drop-down list: Closed and In Process. If the investigation of the case has been completed, or the disease was not confirmed, the Closed option should be selected so that all fields of the form except Case Status become non editable. In Process should be chosen if the investigation of the case has not yet been completed. By default the status is In Process.

Date Entered /
Date Last Saved

The field Date Entered will be populated automatically when the case is created. In the field Date Last Saved the current date will be inserted each time the case is saved. Based upon these two fields, a user will always know when the case was created and when it was last changed. By creation of the case, the values in these fields will be identical. When the form for a new case is opened the Date Entered by default is the current date and the Date Last Saved is empty. Date Entered does not change after subsequent data entry. Date Saved updates each time the case information is saved.

Patient

In the field Patient the last name, first name and middle name of the patient will be displayed. These are taken from the Notification tab. The values in this field will be blank until the completion of the fields Last, First and Middle Name in the Demographic Information group.

Current Diagnosis

The value of the fields Diagnosis or Changed Diagnosis will appear upon the completion of the relevant field on the Notification tab.

Case Classification

The Case Classification field is populated from the field Initial Classification or Final Classification on the Case Investigation tab. Final Classification of the case according the case definition is autopopulated to this field. If final classification is not made an initial case classification remains autopopulated to this field.

Entered By: Officer

The Officer field specifies the employee who entered the notification.

Organization

The Organization field specifies the organization of the person who entered the notification.
Notification tab contains sections to enter general data, demographic and clinical information on the case. The current tab should be filled based on the standard urgent notification form.

Fig. 210 Paper form data section

Field/Variable

Description

Date of Completion
of Paper form

In the field Date of Completion of Paper form the date is entered, when the case investigation paper form is completed. This field is empty by default, its value can be entered in two ways: directly typing in the field (date by default is the current date and time is 12:00:00 AM) or choosing the date from calendar (date by default is the current date).

Local Identifier

In the field Local Identifier a user has to enter the local number assigned to the case in a paper urgent notification form. This field is a free-text field.

Diagnosis

In the field Diagnosis the initial diagnosis at the time of notification has to be specified from a drop-down list. This field is highlighted with a red border and has a yellow background, which means that this field is mandatory. The case could not be saved unless the diagnosis is given.

Diagnosis Date

In the field Diagnosis Date the date of the initial diagnosis has to be specified. This field is blocked and activates when the diagnosis is chosen. Diagnosis Date is empty by default, and its value can be entered in two ways: directly typing in the field (date by default is the current date) or choosing the date from calendar (date by default is the current date).

Fig. 211 Demographic Information section

Field/Variable

Description

Name: Last,
First, Middle.

In the field Last Name the user has to enter the surname of the patient in a free-text form or select it from list by pressing the button. After selecting the patient from list, some fields of the case are autopopulated with data according  to the Person's Details form.
The completion of the field Last Name is mandatory. The case cannot be saved unless the Last Name is given. The fields First and Middle are not mandatory fields, but they can be filled in by name and middle name of the patient in a free-text form. The field Patient in the Case Summary section will be a combination of values from First Name, Last Name and Middle Name fields.

Age/Date of Birth

The age of a patient can be specified in two ways. A user may specify the exact Date of Birth, so that the age value will be calculated automatically, or he may specify the age in the field Age. The second method may be used when the patient’s exact date of birth is unknown. It is not possible to enter manually Age and Date of Birth simultaneously. One of the fields is always un-editable. Date of Birth is empty by default, and its value can be entered in two ways: directly typing in the field (date by default is the current date) or choosing the date from the calendar (date by default is the current date). Age is a numeric field, only numbers can be entered. For the Age the patient age units can also be specified from a drop-down list: days, months, years. By default there is no value selected in this field.

Sex

In the field Sex the sex of the patient can be selected from a drop-down list: Male or Female.

Personal ID Type

In the field Personal ID Type the type of personal ID of the patient can be selected from a drop-down list.

Personal ID

In the field Personal ID Type the personal ID of the patient can be specified. The field is active, only if Personal ID Type is selected.

Region/Rayon

The Region and Rayon information in Current Residence fields group is mandatory. The case could not be saved unless the Region and Rayon are given. These fields can be filled in from the drop-down lists of regions and corresponding rayons. The Rayon cannot be chosen, unless the Region is specified.

Town or Village

The Town or Village field can be filled in from the drop-down list. The Town or Village cannot be chosen, unless the Rayon is specified.

Street

The field Street is a drop-down list. Initially, this drop-down list is empty since it is unnecessary for the system to store the full list of streets in all of the country’s settlements. User can fill in the name of the street in the settlement manually. Street name can be given only if the Town or Village is also defined.

Building/House/Apt.

In the free-form text fields Building/House/Apt. a user can specify the building where the patient lives. Building/House/Apt. info can be given only if the Town or Village field is also filled in.

Postal Сode

A user can fill in the field Postal Сode the same way as the field Street. Postal Сode can be specified only if the Town or Village is defined. The field is editable free-form text.

Phone Number

In the free-form text field Phone Number the telephone number of the patient can be specified.

Citizenship

In the field Citizenship a user can select the nationality or citizenship of the patient. This could be done from a drop-down list. A new value to the drop-down list can be added in the form A01. A user can open this form by clicking button in the field Citizenship.

Longitude/Latitude

The user can enter location longitude and latitude using up-down controls or by the use of Map button (GIS module will be opened).

Longitude is a numerical field. The numbers can be entered in the following format: x.xxxxx and can be entered only in the range of (-180, +180).

Latitude is a numerical field. The numbers are entered in the following format: x.xxxxx and can be entered only in the range of (-85, +85).

If user points a place by the use of the map, Longitude and Latitude are filled automatically.

Name of Employer

The patient’s Name of Employer can be specified. The employer is a place of work, a school, a kindergarten, an institution or other place, which the patient visits regularly interacting with a large number of people. Name of Employer is a free-form text field.

Address of Employer

Address of Employer field opens a separate drop-down window. In this window a user can specify the following:

–Country: by default is the value of the field equal to the country of the current site

–Region: dependent on the chosen Country

–Rayon: dependent on the chosen Region, and is not possible to define unless Region is selected

–Town or Village: dependent on the chosen Rayon, and is not possible to define unless Rayon is selected

–Street and Postal Code and Building/House/Apt.: dependent on the chosen Town or Village, and is not possible to define these fields unless Town or Village is selected.

Date of Last Presence at Work, Study and Preschool Institution

Date of Last Presence at Work, Study and Preschool Institution is empty by default, and its value can be entered by two ways: directly typing in the field (date by default is the current date), or choosing the date from the calendar (date by default is the current date).

Search for Duplicates

Search for Duplicates button allows the search of cases already registered in the system which are similar to the case being created. The cases are compared by the following parameters: initial diagnosis, final diagnosis, local case ID, last name, first name, middle name, patient age, patient age units.

Fig. 212 Clinical Information section

Field/Variable

Description

Date of symptoms onset

In the field Date of symptoms onset a user has to specify the day when the first symptoms were manifested. Date of symptoms onset is empty by default, and its value can be entered in two ways: directly typing in the field (date by default is the current date), or choosing the date from calendar (date by default is the current date).

Status of the Patient at Time of Notification

The field Status of the Patient at Time of Notification can be Dead or Alive. Status could be selected from a drop-down list.

Changed Diagnosis (if known)

The field Changed Diagnosis (if known) should be filled in if the diagnosis was changed during the case’s investigation process. The changed diagnosis can be selected from a drop-down list. If the changed diagnosis was specified, then it is necessary to explain the reason for the change. The reason has to be given in a separate window which appears as soon as a field Changed Diagnosis (If Known) is filled in (form H12). It is not possible to close this window, unless the reason of change is defined. Then the Diagnosis History will be available in a separate window (form H14) by clicking the button in the Changed diagnosis (if known) field.

Date of Changed Diagnosis

The field Date of Changed Diagnosis will remain disabled until a Changed Diagnosis has been selected. If the Changed Diagnosis was specified, than a user can set the date of the change manually. Date of Changed Diagnosis is empty by default, and its value can be entered in two ways: directly typing in the field (date by default is the current date), or choosing the date from calendar (date by default is the current date).

Current Location of Patient

The field Current Location of Patient allows selecting the location of the patient at the time of notification: Home, Hospital, Other. If the patient is in a hospital, the Hospital name can be selected from the drop-down list (otherwise this field is unavailable), if he/she is neither at home nor in a hospital, the Other location name can also be given in the text-field (otherwise this field is unavailable).

Additional Information/Comments Including Movement of Patient During Period of Maximum Incubation

In the text-field Additional Information/Comments Including Movement of Patient During Period of Maximum Incubation additional important data can be entered.

Fig. 213 General Information section with the notification information

Field/Variable

Description

Notification Date

In the field Notification Date, a user specifies the date when the notification was sent. Notification Date is empty by default, and its value can be entered in two ways: directly typing in the field (date by default is the current date), or choosing the date from calendar (date by default is the current date).

Notification sent by /

Notification received by

In the rows Notification sent by/Notification received by the Name and the Facility can be chosen from a drop-down list. The Facility can also be found by clicking the button. A new value can be added to the drop-down lists by clicking the button.
Case Investigation tab includes several sub-tabs related to demographic data, clinical information, sample collection, contact list, case classification (clinical signs), epidemiological links, risks factors and final case classification & outcome. The Case Investigation tab should be filled out based on the completed standard paper Case Investigation form.

Fig. 214 Case Investigation tab - Demografic Information subtab

In the Demographic Information sub-tab the information is already partially autopopulated, based on the data from the Notification tab. These fields are marked with a blue background and a blue border, and are read-only.

 

Field/Variable

Description

Organization Conducting Investigation

The field Organization Conducting Investigation holds data on the organization which will conduct the current case investigation. Organization Conducting Investigation can be chosen from a drop-down list. The Organization can also be found by clicking the button. A new value can be added to the drop-down list by clicking the button.

Starting date of Investigation

In the field Starting Date of Investigation the date and time of the beginning of the investigation can be specified. Starting Date, Time Investigation is empty by default, and its value can be entered in two ways: directly typing in the field (date by default is the current date), or choosing the date from calendar (date by default is the current one). By default the time is 12:00 AM.

Permanent Residence

In the Permanent Residence group a user can enter the permanent registration address of the patient if it differs from the current one.

 

A user can specify:

–The Region from a drop-down list

–The Rayon from a drop-down list which is Region dependent, and not possible to define unless a Region is selected

–The Town or Village from a drop-down list which is Rayon dependent, and not possible to define unless Rayon is selected

–The Street from a drop-down list and Postal Code from a drop-down list and Building/House/Apt. in the text-field which are Town or Village dependent, and not possible to define these fields unless Town or Village is selected

 

The Phone number text-field can also be entered manually. Longitude/Latitude field is filled by using the up-down controls or by the use of Map button (GIS module will be opened).

If the Foreign Address is enabled, the Country can be selected from the drop-down list and Address and Phone number fields are free-text forms.

If Permanent Address same as Current Address is enabled, the Permanent Residence section becomes unavailable.

Address of employer, children’s facility, and school

A user can select the Occupation from a drop-down-list and the business Phone number in a text-field. A new value to the drop-down list of occupations can be added in the form A01. A user can open this form by clicking the button in the field Occupation.

Fig. 215 Case Investigation tab - Clinical Information subtab

Two fields: Diagnosis and Date of Symptoms Onset, on the Clinical Information sub-tab are also already filled in, based on the data from the Notification tab.

 

Field/Variable

Description

Initial Case Classification

The field Initial Case Classification may be populated from a drop-down list with one of three values: Confirmed, Probable, or Suspect.

–Confirmed should be chosen if the case has been confirmed by the confirmatory laboratory tests regardless of having or not clinical symptoms and epidemiological links.

–Probable case that has either clinical and/ or epidemiological link and/or positive result by presumptive test.

–Suspect should be chosen if the case has clinical symptoms only.

 

No new value to the drop-down list of an initial case classification can be added. In the Case Summary section the field Case Classification will be filled with the data entered in the Initial Case Classification field.

Location of Exposure if Known

In the field Location of Exposure if Known a user may specify more precisely the geographical location of the case in the separate form C14. A user can open this form by clicking the ellipsis button in the field Location of Exposure if Known.

Date of Exposure

In the field Date of Exposure the date of the contamination should be specified if it is known. Date of Exposure is empty by default, and its value can be entered in two ways: directly typing in the field (date by default is the current date), or choosing the date from the calendar (date by default is the current date).

Facility Where Patient First Sought Care

In the field Facility Where Patient First Sought Care a user can use the drop-down list to select the name of the organization where the patient asked for a medical assistance for the first time. The facility can also be found by clicking the button. A new value can be added to the drop-down lists by clicking the button. This field is a text field, which could be filled in a free form.

Date Patient First Sought Care

In the field Date Patient First Sought Care a user can enter the date, when the patient asked for a medical assistance for the first time. Date Patient First Sought Care is empty by default, and its value can be entered in two ways: directly typing in the field (date by default is the current date), or choosing the date from the calendar (date by default is the current date).

Non-notifiable diagnosis from facility where patient first sought care

In the field Non-notifiable diagnosis from facility where patient first sought care a user can enter the diagnosis which was stated in the facility where the patient first sought care. The diagnosis is selected from the drop-down list. A new value can be added to the drop-down lists by clicking the button.

Hospitalization

A user can also specify in this tab whether the patient was hospitalized (Hospitalization). If Yes, then when (Date of Hospitalization) and where (free form text field Place of Hospitalization) the hospitalization took place. If the value of the field Hospitalization is No or Unknown, then these 2 fields: Date of Hospitalization and Place of Hospitalization are not available. Date of Hospitalization is empty by default, and its value can be entered by two ways: directly typing in the field (date by default is the current date) or choosing the date from the calendar (date by default is the current date).

Antibiotic/Antiviral therapy administered before samples collection

Below, it is also possible to list any antibiotic/drugs, which had been taken before the samples were collected. In order to enter the information about these medicines the values of the field Antibiotic/Antiviral therapy administered before samples collection has to be Yes, otherwise the Antibiotic table and assigned Add and Remove buttons will be blocked. To add a new record to this table a user has to click Add button, specify the name of the medicine (text-field Name), its dose (text-field Dose) and the date when it was first administered, the Date first administered. This field is empty by default. Its value can be entered in two ways: directly typing in the field (date by default is the current date) or choosing the date from the calendar (date by default is the current date). To remove an antibiotic from the table, a user has to select the corresponding record in the table, and click the Remove button. The program will ask if a user wants to remove the antibiotic. By clicking Yes, the record will be removed, and clicking No, will result in no record will be removed.

Comments

The Comments text-field holds additional information of the patient’s clinical data.

Buttons

Add

Click to add a new record to the table. The value of the field Antibiotic/Antiviral therapy administered before samples collection has to be Yes otherwise the button is not active.

Remove

Click to remove the record from the table. The confirmation window will appear.

Fig. 216 Fig. 217 Case Investigation tab - Samples Collection subtab

In the Sample Collection tab, the information on the collected samples should be entered.

Check-box

Description

Samples Collected

The check-box contains a drop-down list of values: Yes, No, Unknown. By default no value is selected in this field. If the samples were not collected (the Samples Collected field value equals No), the reason must be specified (a new text field Reason appears). In this case the functionality of adding and deleting samples is disabled.

If it is unknown whether the samples were or were not collected (Samples Collected value equals Unknown), no additional fields are available in this tab, and the functionality of adding and deleting samples is disabled.

If the sample is accessioned in by a laboratory then the information on such samples is read-only and non-editable.

If samples were collected (Samples Collected value equals Yes) a user is able to enter comments about these samples. The functionality of adding and deleting samples is enabled.

In order to clear the content of the field Samples Collected the empty field has to be selected. As soon as the empty field is selected, the pop-up window asks if the user wants to clear or not clear the content. If the user confirms the deletion the field is cleared.

To add a new sample, a user has to specify that samples were collected (Samples Collected value equals Yes) and click the Click here to add a new row line. A new record will be created in the table.

Filter samples by Diagnisis

The check-box aimed to filter the list of sample types in the grid below. The default value copies the value of the Filter samples by Diagnosis check-box the the System Prefernces.

Yes - The list in the Sample Type field in the table below will contain only samples applicable bor the diagnosis of the case in accordance with the Diagnosis -> Sample Type matrix.

No - The list is not filtered and contains all sample types.

Field/Variable

Description

Sample Type

Sample Type field defines the type of the collected sample; The value of this field is selected from a drop-down list.

Local Sample ID

Local Sample ID text-field contains an internal ID of the sample assigned to it during the collection process. This free-text field is filled in manually.

Collection Date

Collection Date field holds data about the date of the sample’s collection. The record creation date is in this field by default, but it could be changed manually by directly typing in the field or choosing the date from the calendar.

Sent Date

Sent Date field displays the date when the sample was sent to the laboratory. A user has to set it manually, but it could not be earlier, than the sample Collection Date. The Sent Date field is empty by default, and its value can be entered in two ways: directly typing in the field (date by default is the current date), or choosing the date from the calendar (date by default is the current date).

Accession Date

Accession Date field defines when the sample was received by a laboratory. It is a non-editable field, which is filled in automatically after the sample is Accessioned In. By default the value of this field equals to the date of accession in the system, but a laboratory employee can select from a calendar or enter manually the a different date during the process of accessioning this sample.

Sample Condition Received

Sample Condition is selected from the drop-down list after the sample is received by the laboratory.

Comment

Text-field value can be any comment connected to the quality of the received sample.

Test Name

Test Name field contains the name of the assigned test. If several tests were assigned to one sample, then for this field, the name of the main test can be chosen from a drop-down list of assigned tests. By default this field is blank. The ability to choose a test name is available here only after the corresponding test results are registered in database.

Test Result

Test Result field states the result of the conducted test and is a non-editable field which depends on the selected Test Name.

Result Date

Result Date field specifies the date when the test result was entered and is a non-editable field which depends on the selected Test Name.

Collected by Institution

The Institution field specifies the organization which collected the samples.

Sent To Organization

Specify the facility where the collected sample is to be investigated. This value will be used in the Laboratory module to distinguish samples between offices. Even if the Sent To Organization field is not mandatory, it is highly recommended to fill it out.

Additional Test Requested and Sample Notes

Free-form text field where any additional comments can be entered.

 

Fig. 218 Case Investigation tab - Contact List subtab

In the Contact List tab, a user can specify the persons, who were in contact with the patient, and who could have infected or have been infected by the patient.

To add a contact to the list, a user has to click the Add Contact button, otherwise the contact table is empty.

If there is no contact in the table then the Edit Contact and Remove Contact buttons are blocked. If there is only one saved contact in the table then the Remove Contact button is also blocked.

As soon as the Add Contact button is clicked a new empty row appears in the table.

 

Field/Variable

Description

Name

A contact person can be chosen from the existing list of people in the system by selecting the magnifier tool in the field Name (form H03). Or a new contact can be created by pressing the button with the ellipsis in the field Name (form H04).

Relation

Relation field specifies the status of the relationship between the patient and contact person. The relation could be chosen from a drop-down list: Classmate, Coworker, Family, Friend or Neighbor. By default the value of this column equals Family.

Date of last contact

Date of last contact field value is the date of the last contact of the patient with this person. This field is empty by default, and its value can be entered in two ways: directly typing in the field (date by default is the current date) or choosing the date from the calendar (date by default is the current date).

Place of last contact

Place of last contact free-text field contains the name of the place where the last contact took place.

Contact’s information

Contact’s information field defines the place of the contact’s current residence taken from the Contacts’ data. It is not possible to enter data directly in this field.

Comments

In the free-text field Comment, a user can provide his/her comments in a free form.

Buttons

Add Contact

Click to add a new record to the table.

Edit Contact

Click to edit the selected contact. The Person Details form will open.

Remove Contact

Click to remove the record from the table.

A single person can't be added to the Contact List twice. If two records are considered equal, then a notification window informs a user that a record already exists and does not allow the creation of another record.

The check of records’ similarity is performed every time, when a user selects a person from Persons List (form H03).

Fig. 219 Case Investigation tab - case classification subtab, view for botulism; for other diseases it has another view

The patient’s symptoms have to be inserted in Case Classification tab. The set of fields shown in this sub-tab will depend on the supposed diagnosis of the patient that is defined in the Notification tab. If no diagnosis is specified then this sub-tab is empty. The view of this tab is specified in the flexible forms designer (System/Configuration/Flexible Forms Designer) in Templates Editor of Human Clinical Signs. If there is no template defined for this specific diagnosis, then a unified template is applied. The user clicks on the Add Parameter button to add a parameter, and clicks on the Remove Parameter button to delete a dynamically added parameter.

Fig. 220 Case Investigation tab - Epidemiological Links and Risk Factors. View for botulism; for other diseases it has another view

Epidemiological Links and Risk Factors sub-tab specifies the additional data which is important to establish a correct diagnosis. For each diagnosis a corresponding set of fields will be displayed. These fields will be relevant to this particular diagnosis. If no diagnosis is specified, then this sub-tab is empty. The view of this tab is specified in the flexible forms designer (System/Configuration/Flexible Forms Designer) in Templates Editor of Human Epi Investigations. If there is no template defined for this specific diagnosis, then a unified template is applied.

Fig. 221 Case Investigation tab - Final Case Classification and Outcome subtab

Final Case Classification and Outcome tab is filled in when the the final case classification and outcome is known and also when the case is linked to an Outbreak.

 

Field/Variable

Description

Final Case Classification

The field Final Case Classification can be populated with one of five values from a drop-down list: Confirmed, Probable, Lost to Follow-up, Not a Case, or Suspect.

–Confirmed should be chosen if the case has been confirmed by the confirmatory laboratory tests regardless of having or not clinical symptoms and epidemiological links.

–Probable case that has either clinical and/ or epidemiological link and/or positive result by presumptive test.

–Suspect should be chosen if the case has clinical symptoms only.

–Lost to Follow-up - the case investigation failed because of its tracking is difficult or impossible.

–Not a Case - the case confirmation failed.

Date of Final Case Classification

The date of final case classification is specified in this field. The field becomes editable after the Final Case Classification is specified. The value can be entered in two ways: directly typing in the field (date by default is the current date), or choosing the date from the calendar (date by default is the current date).

Final Diagnosis / Date of Final Diagnosis

The fields Final Diagnosis and Date of Final Diagnosis will be filled in with the data from the Notification tab. These fields are non-editable. If initial diagnosis was changed, then the Final Diagnosis value will be taken from Changed Diagnosis (if known), and the Date of Final Diagnosis value will be taken from the Date of changed diagnosis field. If the initial diagnosis was not changed, then the Final Diagnosis value will be taken from Diagnosis, and the Date of Final Diagnosis value will be taken from the Diagnosis Date field.

Basis of Diagnosis

The basis of diagnosis is specified using the following check-boxes: Clinical, Epidemiological Links, Laboratory Test.

Outcome

In the field Outcome a user can select one of three case outcomes from a drop-down list: a patient Recovered, Died, or his/her state is Unknown. By default in this field no value is selected.

Date of Discharge

In the field Date of Discharge the date when the patient discharged from the hospital has to be specified. This field is empty by default, and its value can be entered in two ways: directly typing in the field (date by default is the current date) or choosing the date from the calendar (date by default is the current date). The field is editable only if Outcome field value is Recovered.

Date of Death

In the field Date of Death the date when the patient died has to be specified. This field is empty by default, and its value can be entered in two ways: directly typing in the field (date by default is the current date), or choosing the date from the calendar (date by default is the current date). The field is editable only if Outcome field value is Died.

Is this case related to an outbreak

The Is this case related to an outbreak field indicates if the current case is related to an outbreak. One of three values can be selected from a drop-down list: Yes, No, Unknown. By default no value is selected in this field.

Outbreak ID

The Outbreak ID field is blocked by default. If the case relates to an outbreak (Is this case related to an outbreak value equals Yes), the Outbreak ID field becomes activated and can be filled in, otherwise this field is blocked.

 

The Outbreak ID can be selected from the list of outbreaks in the system (magnifier tool – form C10) or a new outbreak can be created (ellipses tool – form C11).

Comments

In the free-text field Comments, a user can provide his/her comments in a free form.

Epidemiologist Name

In the free-text field Epidemiologist Name the name of the epidemiologist, who finally classified the case, could be given.

Tests tab is designed to track if tests were conducted, view all tests entered by laboratory technicians (planned, being performed and completed), and to register test results manually directly in the case. Also this tab contains a test results summary and interpretation. For derivatives the Sample Type column contains the type of the original sample.

Fig. 222 Tests tab

In this tab the information on the performed tests should be entered. The Tests Conducted field contains a drop-down list of values: Yes, No, and Unknown). If testing is done in the internal laboratory that has EIDSS this field is autopopulated with "Yes" option from Laboratory module. If test is conducted at the external laboratory user has to enter this field manually.

If the tests were performed at an off-site laboratory then the information for such tests is read-only and non-editable.

In order to clear the content of the Tests Conducted field, the blank field has to be selected. As soon as the empty field is selected, the pop-up window asks if the user wants to clear or not clear the content. If the user confirms the deletion, the field is cleared.

To add a test a user has to specify that the tests were conducted (Tests Conducted value equals Yes), and click the Click here to add a new row line. A new record will be created in the table, and the Local Sample field will be automatically expanded.

Note: new tests can be added only by the user with proper access rights.

Field/Variable

Description

Tests Conducted

Displays if any tests were conducted or not conducted.

Lab Sample ID

Filled automatically with Lab Sample ID. If derivative or aliquot was created its own lab ID will be displayed.

Sample Type

Filled automatically with Sample Type. If derivative was created, parent Sample Type is displayed.

Local Sample ID

A user can choose a Local Sample ID from a drop-down list, or can enter the value manually.

Test Name

Test name, assigned to the chosen sample.

Test Diagnosis

Diagnosis, which can be validated by the selected test.

Test Run ID

Non-editable field.

Result Date

Result Date field specifies the date when the test result was entered and is a non-editable field which depends on the selected Test Name.

Functional Area

Filled in automatically if it is defined for the accessioned in sample.

Test Category

Test category can be entered from the drop-down list which includes: Confirmatory, Presumptive, Procedural.

Test Status

Filled automatically with Test Status.

Result/Observation

Result of the chosen test for the chosen sample.

Buttons

Delete

To delete test result, select it and click Delete button. Warning message withYes/No confirmation will appear.

Note: only manually created tests can be deleted. It is impossible to delete a test if it was performed at the off-site laboratory.

If "n" tests were performed with one sample than there will be "n" rows in the table with the same Lab Sample ID, but different Test Names.

Results Summary and Interpretation group is used by epidemiologist to enter the interpretation of received test results. The New button is clicked to add a new record to the table and it is active only if Completed test is selected in Tests Conducted section. When there are no records in the table, the Delete button is blocked. When a new record is added, the Delete button is active, and the New button is blocked until the Diagnosis and Rule In/Rule Out fields are filled in. It is not possible to interpret the test, which has no results and is not Completed – both New and Delete buttons are blocked. If the record was saved, then it is not possible to delete it - Delete button is blocked.

 

Field/Variable

Description

Diagnosis

Diagnosis column allows a new diagnosis to be chosen which could be stated based on the received result of each specific test. The diagnosis is autopopulated from the Test Diagnosis field but can be changed by selecting a different diagnosis from the drop-down list.

Test Name and Test Category

Test Name and Test Category are filled in automatically from the Tests group for each specific test. These fields are non-editable fields. Drop down menu of Test Category includes: Confirmatory, Presumptive, Procedural.

Rule Out/Rule In

Rule Out/Rule In column defines if the diagnosis is approved or denied based on a test result. The value of the fields has to be chosen from a drop-down list: Rule In, Rule Out.

Comments (Rule Out/Rule In)

Comments about the decision to rule in/out are entered here.

Date Interpreted

Date Interpreted column stores the date when the diagnosis based on the result was ruled in or out; by default it is the date of adding the record to the table. This field is a non-editable field.

Interpreted by

Interpreted by column is filled in automatically as soon as the value of the Rule Out/Rule In column is chosen. The value of this column is the name of the user, who interpreted this particular result.

Validated (Y/N)

Validated (Y/N) column checks if the interpretation is validated or not. Only users, who can execute the function “Can validate Test Result Interpretation” will be able to select the validation check-box.

Comments (Validated)

Comments about validation can be entered here.

Date Validated

Date Validated column stores the date when the interpretation was validated; by default it is the date of adding the record to the table. The field is non-editable.

Validated by

Validated by column shows the user, who validated the interpretation. Filled in automatically and non-editable.

Buttons

New

Click to add the new record to the table. The button is blocked until the Diagnosis and Result/Observation fields are filled in.

Delete

Click to delete the record from the table.

Paper Forms button allows to open a Case Tests paper form, connected to the tests of the current human case.
