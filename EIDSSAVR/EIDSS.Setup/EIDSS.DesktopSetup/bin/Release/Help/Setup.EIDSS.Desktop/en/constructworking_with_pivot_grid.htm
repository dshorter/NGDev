Pivot grid

A pivot grid is a data-summarizing tool. For typical data entry and storage, data is usually flat. Flat is defined as consisting of only columns and rows. While there is a lot of information stored in such data, it can be difficult to receive a summarized information. A pivot grid can help quickly summarize the flat data, giving it depth, and receive the desired information. The usage of a pivot table is extremely broad and depends on the situation. The first question to ask is "What am I looking for?" In the example here, let us ask "How many human cases do we have for each region and rayon for each year?"

Fig. 540 Pivot grid example 2

A pivot table usually consists of row, column, and data area. In the example above, the columns are Entry Dates grouped by a year, the rows are Current Residence of Patient - Regions and Rayons, and the data wanted in the table cells is Case ID (Human Case - Case ID). These fields were dragged onto the pivot table from the Grid Fields Customization pane.

Pivot tables also allow to execute several types of aggregate functions, including: sum, average, count, etc. (see the Apply aggregate functions section) to be applied to the data dragged to the Data Area. In the example where a number of cases is required, a Count aggregation function is used. In the example where the density of  cases (according to the population) is required, use the administrative unit (Rayon or Region) field in the rows, and use Rates per 10000 or per 100000 aggregate functions.
