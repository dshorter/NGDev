Laboratory Management Overview

A sample life cycle starts with a sample accession in, ends with sample destruction, and contains the following stages:

1.Accession In. A sample record is added to EIDSS by initial creation or by acceptance from another organization. At this moment it is required to enter the basic data of the sample: laboratory sample ID, which is entered manually or auto-assigned in accordance with the Unique Numbering Schema, its condition at the moment of accession, storage location, etc. Besides this information, enter its location in the EIDSS repository, accession-in date, etc. A sample to be accessioned in can be either already linked to a case/session or not yet linked, in the first case some information from the case/session will be automatically used for the sample. After this operation, the sample obtains the Accessioned In status.

More details regarding accession in, read the following sections:

·Accession in existing sample, linked to case/session (IF)

·Register and accession in sample, not linked to case/session (LS)

·Accession in existing sample(s), linked to case/session (LS)

·Group accession In (IF)

·Group accession In (LS)

2.Creation of Aliquots and/or Derivatives. A sample with an Accessioned In status can be used to create some aliquots and/or derivatives if necessary. However, this step is not mandatory and can be skipped. The newly created samples (aliquots or derivatives) receive the Accessioned In status; the status of the parent sample is not changed.

For details, read the following sections:

•Create aliquot (IF)

•Create aliquot (LS)

•Create derivative (IF)

•Create derivative (LS)

3.Assigning Tests. After a sample is accessioned in, tests can be planned . If the sample is linked to a case or session, the case/session diagnosis defines the list of available tests. If neither case nor session is linked, the complete list of tests is available. The tests obtain the Not started status; the status of the sample is not changed.

For details, read the following sections.

•Assign tests during sample accession in (IF)

•Assign tests in Samples List form (IF)

•Assign tests from the Sample form (IF)

•Assign test (LS)

4.Doing Tests. After a test is assigned, it can be initiated. To do this, change its status to In progress; i.e., the test is started but not yet finished. This step can be omitted if a test assignment is registered when its result is already known and can be entered.

For more details, read the Start test (IF) and Start tests (LS) sections.

5.Entry of Test Result. Once a test is done, enter its result to EIDSS.

•If a test result is not yet validated (finalized), then enter it with the Laboratory Section and set the test status to Preliminary. In this case the test validation needs to be recorded as an additional step.

For details, read the Enter test result (LS) section.

•If the result is already validated (finalized), then enter it either with 2 steps in the Laboratory Section form (as described above), or with one step in the Sample form. Once the test is validated, its status is changed to Final.

For details, read the Enter test result (LS) section.

•If a test was performed in an external laboratory, then the result shall be entered as an external test result.

For details, read the Set result for test, performed in external laboratory (IF) and Set result for test, performed in external laboratory (LS) sections.

6.Test Result Validation. Once a test result is entered, it shall be validated (confirmed) by an authorized person if it was not done before. After validation, the test obtains the Final status.

For details, read the Validate test result (LS) section.

7.Test Result Amendment (Rejection). If a test result is validated but appears to be wrong, the test result has to be amended. In this case the test obtains the Amended status. The step is not mandatory and is usually skipped. The Amended status is a dead-end for a test; only another amendment or a routine destruction can be planned for such a test. To repeat the amended test, assign a new test instead.

For details, read the Amend test result (IF) and Amend test result (LS) sections.

8.Sample Destruction. Once all tests are completed and validated, a sample can be first marked for destruction (the status is Marked for Routine Destruction) and then destroyed (the status is Destroyed). After the destruction, the sample is kept in the system and can be found via a search.

For details, read the following sections:

•Mark sample for disposition (destruction) (IF)

•Mark sample for routine destruction (LS)

•Destroy sample (IF)

•Destroy sample (LS)

Moreover, any sample with an Accessioned In status which has no tests with an In Progress status can be transferred out to another organization for external laboratory tests, research, etc. For details, read the Transfer out a sample (IF) and Transfer out a sample (LS) sections.

Once tests are performed on the sample, their results can be registered in the system. For details, read the Set result for test, performed in external laboratory (IF) and Set result for test, performed in external laboratory (LS) sections.

A transferred out test can be transferred back in. For details, read the Accession in/ transfer in existing sample(s), linked to case/session (LS) and Transfer in a sample sections.

Tests can be processed within batches, which are sets of tests combined to perform the same test. Usually it reflects a test tube holder used for a simultaneous test performance. For details, read the following sections: Work with batches, Create batch in Tests List form, Create batch in Batches List form, Finalize a batch (IF).
