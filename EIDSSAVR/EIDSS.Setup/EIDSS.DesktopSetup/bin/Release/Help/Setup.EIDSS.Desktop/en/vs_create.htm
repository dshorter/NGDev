Create vector surveillance (VS) session

Information about a vector surveillance session can be registered in summary or with much detail. Detailed information consists of pools/vectors, collected during the session, the area and time of their collection, and the name of the person who collected the material. Define the samples obtained from the collected material once the information about collected material is registered. The following steps are registration of field and laboratory tests performed for the collected material. All steps mentioned above are performed in the Vector Surveillance module except for the laboratory tests, which are conducted in the Laboratory module and only displayed in the Vector Surveillance module.

All samples including registered samples in the Vector Surveillance module are available in the Laboratory module.

Summarized information is very brief information containing data about collected vectors, the location where they were collected and diseases confirmed by tests performed for the collected material. The information about tests: number, type, and date are not registered, only the confirmed diseases are registered.

 

1.Do one of the following:

•From the Create menu, select Vector Surveillance Session.

•From the Journals menu, select Vector Surveillance Sessions. The Vector Surveillance Sessions List window opens. Click the New button.

•Click the New Vector Surveillance Session on the menu toolbar.

The Vector Surveillance Session window opens.

2.Enter general information of VS.

3.Enter information about the session. This can be one of the following:

•Summary Information

•Detailed Information

•Register vectors/pools

•Register general information for vectors/pools

•Register specific information for vector/pool

•Register samples for vector/pool

•Register fields test results for vector/pool

The list of VS session diseases is comprised of all diseases confirmed by tests (field or laboratory) performed for vector/ pools collected within the session.

4.Link VS session to an outbreak if necessary.

5.Save VS session.

6.Close VS session.

 

The replication of the saved vector surveillance session will commence and it will be forwarded to the EIDSS server and other subsystems.
