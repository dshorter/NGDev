Accession In L05 form for AS Session

The Accession In L05 form view depends on the type of the case or session related to the sample. In fact, the Sample Details and Additional Test Requested and Sample Notes sections are the same, and described in the Accession In L05 section. Only the General information and Sample Receiving/Registration sections differ. Their view for a sample linked to an active surveillance session is described below.

Fig. 486 General information and  Sample Receiving/Registration panes

The fields of the General information pane are not available except for the Session ID field where another session can be selected to reload the entire form.

The availability of the fields in the Sample Receiving/Registration pane depends on the status of the sample. For a newly created sample all fields are available except for the Tests field, but only until the form is saved. Once the save  occurs, all fields become read-only.

For an existing sample during its accession in process the following fields are editable: Lab Sample ID, Accession Date, Sample Condition Received, Comment, Storage Location, and Functional Area. The fields are available until the form is saved for the first time, afterwards they become read-only as well.

All fields of the samples with an Accessioned In status are read-only.

 

General Information Fields

Session ID

The unique ID of the session. To open another session:

1.Click the Search button (). The Active Surveillance Session V18 form opens.

2.Either select a necessary session in the grid and click the Select button, or double-click it.

After a session is selected, the search form closes. The session details shall automatically populate the fields of the General information pane.

To view the session details, click the Details () button. The Active Surveillance Session V18 form opens.

Session Status

The status of the session.

Campaign ID

The unique ID of the campaign related to the session.

Campaign Name

The name of the campaign related to the session.

Campaign Type

The type of the campaign related to the session.

Sample Receiving/Registration Fields

Sample Type

Select a type for a new sample. The type of an existing sample is not available for a change.

Tests

The number of tests assigned to the sample.

Field Sample ID

Enter the field ID of a newly created sample. For an existing sample a previously specified field sample ID is displayed and cannot be changed.

Animal ID

Select an animal or avian from which the sample is collected. The field is editable for a newly created sample only.

Species

Select the species of the animal or avian. The field is editable for a newly created sample only.

Lab Sample ID

A laboratory ID of a sample is automatically created for a new sample and can be manually changed until the form is saved for the first time. For an existing sample, the field is read-only.

Collection Date

Enter the date when a sample was collected. Available for newly created samples only; for those accessioned before, the field is read-only.

Accession Date

Specify the date when the sample was accessioned. The current date is used by default and can be changed. For accessioned in samples, the field is read-only.

Sample Condition Received

Specify the condition of the sample at the moment of accession. If the condition differs from a good condition, an explanation is required in the Comment column.

After the sample is saved, the field becomes unavailable.

Comment

Comments are displayed. For a rejected sample or a sample accepted in with a poor condition status, the field is mandatory and requires an explanation.

Storage Location

Select a storage location for a newly accessioned sample. The field is available until the form is saved for the first time after the sample's accession.

Functional Area

Select a department of the organization where the sample is stored and analyzed. This applicable only for those samples accessioned in with a good or poor conditions status.

Buttons

Search

Finds and adds the list samples by their complete local or field ID entered in the Search by local/field Sample ID field.

1.Enter the sample's local or field sample ID in the Search by local/field Sample ID field.

2.Click the Search button. All samples, that have the entered local/field sample ID are added to the list.

Print Barcodes

Prints barcode labels for accessioned samples. Only labels for samples accessioned in with a good or bad condition status can be printed.

1.Save the form.

2.Select an already accessioned and not rejected sample.

3.Click the Print Barcodes button. The Print Barcode Labels form with barcode preview opens.

4.Click the Print button ().

Accession

Accessions selected samples. Only non-accessioned samples can be accessioned.

1.Select a non-accessioned sample in the grid.

2.Click the Accession button. The selected record becomes editable.

New

Creates a new sample. For details, read the Accession in sample, linked or not linked to case/session (IF) section.

Delete

Deletes selected samples. Only those records that were added through the current Accession In form and never saved can be deleted. To delete a sample, select it and click the Delete button.

Select Test

Assigns a test for the selected sample. For details, read the Assign tests during sample accession in (IF) section.

1.Click on the sample in the list.

2.Click the Select Test button. The Select Test L20 form opens.
