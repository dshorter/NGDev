Accession in group of samples (IF)

A group of any samples linked to any cases or sessions (the same or different) can be accessioned at the same time if they have the same Local/Field Sample ID. Only samples with the Unaccessioned status can be accessioned. This operation is available only if the Lab Module Simplified Mode check-box is clear in the System Preferences.

 

1.From the Journals menu, select Samples. The Samples List window opens.

2.Click the Accession button and from the drop-down menu select Group Accession In.

Fig. 443 Group Accession In

3.Enter the sample field/local ID used for the entire group to the Scan sample field/local ID here field manually or with a scanner and click the Add button. The Select sample window opens with the list of samples that have the same sample field/local ID.

Fig. 444 Select sample

4.Select the samples to be accessioned and click the Select button. The Select sample window closes. The selected samples are added to the Group Accession In window.

5.Repeat steps 3 and 4 to add all samples to be accessioned to the Group Accession In window.

Fig. 445 Group Accession In window

6.Once all desired samples are in the list, click the Accession All button. The samples are updated as follows:

•The current date populated the Accession Date cell and can be changed.

•New Barcode (lab sample ID) is automatically assigned.

•The Sample Condition Received cell is set to Accepted in Good Condition and can be changed if necessary. If the value is changed, enter an explanation to the Comment cell. If the Sample Condition Received field value is changed to Rejected and saved, then the sample is hidden and cannot be found by a search

7.Select the Storage Location for each sample if required.

8.Print new barcode labels for the samples by clicking on the Print Barcodes button.

9.Save and close the Group Accession In window.

The samples are accessioned in. The changes made in the case/session are replicated to other EIDSS sites in accordance with the filtration rules.
