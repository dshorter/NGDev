The Active Surveillance Session window:

Session Summary section remains displayed when moving through the tabs below. It contains the basic information about the session.

Fig. 354 Session Summary section

Field/Variable

Description

Session ID

The field contains the unique number of the session and will be populated when the session is saved for the first time. The Session ID is generated according to the Unique Numbering Schema.

Session Status

Define the session status as Open or Closed.

Session Start/End Date

Set the start and end dates of the session.

Campaign ID

This attribute is used to link a session to a certain campaign. An already existing campaign can be chosen by clicking the button which opens the search form. Also a new campaign can entered by clicking the button which opens the campaign form.

Campaign Name

Name of the linked campaign.

Campaign Type

Type of the linked campaign.

Site

Site unique number where the case was first created.

Officer

Name of the user who first saved the case.

Date

Date of first save.

Fig. 355 Address section

Description

Country

Name of the country where the session was held.

Region

Name of the region where the session was held. Only regions of selected country are displayed in drop-down list.

Rayon

Name of the rayon where the session was held. Only rayons of selected region are displayed in drop-down list.

Town or Village

Name of the town or village where the session was held.. Only towns and villages of selected rayon are displayed in drop-down list.

Detailed Info tab consists of two subtabs: Farm/Animal/Samples and Tests.

 

Farm/Animal/Samples sub-tab can be viewed in two different modes: Standard View Mode and Grid Mode. To switch between these two modes use the Table view check-box.

Standard View Mode displays four sections: Farms, Herd/Species Info, Animals and Sample Receiving/Registration.

 

Fig. 357 Farm/Animal/Samples subtab - Standard view mode

Farms section displays information about the farm(s) related to the current session.

Field/Variable

Description

Farm ID

Farm ID is displayed.

Farm Name

Farm name is displayed.

Farm Owner

Farm Owner name (Last Name, First Name, Middle Name limited by space) is displayed.

Ownership Structure

Ownership structure of the selected farm is displayed.

Buttons

Select

Press to select the existing farm from Farms (V04) window. The Farms section will be filled automatically according to the information of the selected farm.

New

Press to open the Farm Info Details (V06) window, and create a new farm.

Delete

Press to delete the selected farm. A confirmation window with Yes/No buttons appears: “The object will be deleted. Delete object?”

Edit

Press to open the Farm Info Details (V06) window and edit the farm.

Herd/Species Info section displays information about the herd and species on the selected farm.

Field/Variable

Description

Farm/Herd/Species

Select the species from the drop-down list or click the button to add another species to the Species reference editor. One species can be specified for one herd only once, but the herd can hold different kind of species.

Total

Column Total is designed to enter the total number of animals of certain species in the herd. To enter number of animals, use the cell in Species row. This cell has integer type and displays like a text-box. The number of species in every herd and farm will be calculated automatically.

Note (Include breed)

Column Note is designed to enter comments for species. It is a string attribute with memo-box control.

Buttons

New Herd

Click the New Herd button, and a new herd record will be added.

New Species

Select a herd row in the grid to add a species, and click the New Species button. New species record will be added for selected herd.

This button is active only if herd or species row selected.

Delete

Press to delete the selected item. Conformation window with Yes/No buttons appears: “The object will be deleted. Delete object?”

Results Summary and Interpretation section:

Field/Variable

Description

Diagnosis

Filled automatically according to a selected test in test table.

Test Name

Filled automatically according to a selected test in test table.

Test Category

Filled automatically according to a selected test in test table.

Rule Out/ Rule In

This column contains a drop-down list with the Rule in Value for Test Validation reference. The Rule In value can be specified if the test confirms the selected diagnosis. The Rule Out value can be specified if test does not confirm the selected diagnosis.

Comments

Enter comments.

Date Interpreted

Filled automatically with the current date when Rule Out/ Rule In is specified.

Interpreted by

Filled automatically with the current user when Rule Out/ Rule In is specified.

Validated (Y/N)

This column contains a check-box element. It has mixed value by default: the diagnosis is neither declined nor validated by officer who has permission for the validation. If check-box is selected, then the diagnosis is validated by the officer, if unselected, then the diagnosis is declined.

Comments

Comments are entered here.

Date Validated

Is filled automatically with current date if Validated (Y/N)=Yes.

Validated by

Is filled automatically with current user if Validated (Y/N)=Yes.

Buttons

New

Click the New button to add a new interpretation. The new record will be added to the table.

Delete

To delete, select the interpretation, and click the Delete button. A warning message with a Yes/No confirmation appears.

Once saved, the interpretation cannot be deleted.

The Summary Info tab is designed to record the overall information about the current session. The summary information does not rely on other information of the session, it is filled in independently from Detailed Info tab, therefore, it represents an independent source of information. Summary Info tab usually filled in when there is no need to capture detailed information about all the animals, studied during the session, and only summary information about the session is needed.

Create Case button is used to create a new Veterinary Case from the Active Surveillance Session. After clicking the Create Case button, the system checks if there is at least one record in the AS Session, which has Rule In interpretation and for which a case has not been created yet. If there are no such records, a window appears with the following message "There are no records in the session with Rule In interpretation, for which a case has not been created yet.".
