The Outbreak Details window:

General Info tab displays general information about outbreak.

Data fields

Outbreak ID

The Outbreak ID field displays the unique number of the outbreak. This number is populated when the case is initially saved.

Start Date

The Start date field specifies the date when the outbreak began.

End Date

The End date field specifies the date when the outbreak ended.

Diagnosis/Diagnoses Group

The Diagnosis/Diagnoses Group field specifies the diagnosis or diagnosis group of an outbreak. A specific diagnosis or the entire diagnoses group should be selected from the drop-down list. The field is mandatory.

Outbreak Status

The Outbreak Status field specifies the status of an outbreak. The following statuses may be selected from the drop-down list:

Confirmed - the outbreak was confirmed by epidemiologic and clinical features, as well as by laboratory

Probable - the information about  the outbreak is available only from the farm owners and has not been confirmed clinically or by laboratory tests

Suspect -  the outbreak was confirmed by epidemiologic and clinical data, but has not been confirmed by the laboratory

Not an Outbreak -  the outbreak was not confirmed.

Primary Outbreak Location

The Primary Outbreak Location field specifies the geographic location of an outbreak. Click the button, and the drop-down window with Geographic Location form is expanded.

Primary Case/Session

The main case or session of an outbreak.

Description

The Description field may specify a text description of an outbreak.

 

Related Case/Session Reports section contains the list of cases and sessions, related to the outbreak, and includes the following parameters:

Data fields

Case/Session ID

Case or session ID is displayed.

Case/Session Date

Case’s date is displayed.

Human Case: If the value of the Date of symptoms onset field is not blank, then this date is displayed;

If the value of the Date of symptoms onset field is blank, and the value of the Notification date field is not blank, then the notification date is displayed.

If the values of the Date of symptoms onset and Notification date fields are blank, then the value of the Date Entered field is displayed.

Vet Case: If the value of the Investigation Date field is not blank, then this date is displayed. If the value of the Investigation Date field is blank, then Data Entry Date is displayed.

Vector Surveillance Session: The Start Date is displayed.

Source of Case/Session Date

The name of the field of a case/session is displayed which determines the date of the case/session.

Case Classification

Case’s classification is displayed.

Human Case: If the Final Case Classification field is not filled in, then the Initial Case Classification is displayed, otherwise Final Case Classification is shown.

Vet Case: Case Classification is displayed.

Vector Surveillance Session: Case Classification is blank.

Diagnosis

Case’s Diagnosis is displayed.

Human Case: if Changed Diagnosis is not filled in, then Diagnosis is displayed, otherwise Changed Diagnosis is shown.

Vet Case: If the Final Diagnosis field is not filled in, then the last entered Tentative diagnosis is displayed, otherwise Final Diagnosis is shown.

Vector Surveillance Session: Disease is displayed.

Exposure Location

A case/session location is displayed as it is defined in the Format of Geo Location.

Human Case: Location of Exposure

Vet Case: Farm Location.

Vector Surveillance Session: Collection Location.

Address

A case/session location is displayed as it is defined in the Format of Address.

Human Case: Current Residence

Vet Case: Farm Address.

Vector Surveillance Session: Collection Location.

Patient/Farm Owner

Name of the patient or the farm owner is displayed in this field.

Related Case/Session Reports section contains the following functional buttons:

Buttons

Mark As Primary

Select a case or session and click this button to mark the selection as a primary case/session for the current outbreak.

Add Case/Session -> Add Veterinary Case

Click on the Add Veterinary Case button, the Veterinary Cases List in selection mode is opened in a new window. A user may select a case by clicking on the Select button. The selected case is added to the Related Case Reports section of the Outbreak Details form.

If the selected case has already been attached to an outbreak, it will be offered to remove the case from its current outbreak and link to the outbreak being edited.

If a user clicks on the Yes button, the case is removed from an old outbreak and attached to the current outbreak. Click on the No button, nothing happens, and the selection is canceled.

The Add Veterinary Case button is available if a user has an access to the system function Access to Veterinary Cases Data (at least READ right).

Note: if the Outbreak diagnosis or diagnosis group is defined, it is impossible to add a case with different diagnosis.

Add Case/Session -> Add Human Case

Click on the Add Human Case button, the Human Cases List in selection mode is opened in a new window. A user may select a case by clicking on the Select button. The selected case is added to the Related Case Reports section of the Outbreak Details form.

If the selected case has already been attached to an outbreak, it will be offered to remove the case from its current outbreak and link to the outbreak being edited.

A human case is possible to add to an outbreak only if the value of the field Is this case related to an outbreak is blank or Yes. If the value is No, the case cannot not added to an outbreak.

The Add Human Case button is available if a user has access to the system function Access to Human Cases Data (at least READ right).

Note: if the Outbreak diagnosis or diagnosis group is defined, it is impossible to add a case with different diagnosis.

Add Case/Session -> Add VS Session

Click on the Add VS Session button, the Vector Surveillance Sessions List in selection mode is opened in a new window. A user may select a session by clicking on the Select button. The selected session is added to the Related Case Reports section of the Outbreak Details form.

If the selected case has already been attached to an outbreak, it will be offered to remove the case from its current outbreak and link to the outbreak being edited.

Note: if the Outbreak diagnosis or diagnosis group is defined, it is impossible to add a session with different diagnosis.

View Details

If a user selects a case/session in the Related Case Reports section, and double-clicks on this case/session or clicks on the View Details button, then the selected case is opened in a new Human Case Report Form (for human case), Investigation Form for Avian Mortality (for avian case), Investigation Form for Livestock Diseases (for livestock case) or Vector Surveillance Session form (for vector surveillance session).

Depending on the user’s access to the system functions: the forms is opened in read, edit and/or delete modes.

Remove

If a user selects a case in the Related Case Reports section and clicks on the Remove button, then the selected case is removed from the outbreak. The corresponding field in the case Outbreak ID is cleared up.
Notes tab is used to record any notes related to the outbreak.

Fig. 395 Notes tab

Data fields

Note

The notes to an outbreak may be entered in the Note field. In order to confirm a note, click OK button.

In order to cancel the creation of a note or a change of note, click the Cancel button.

The OK button is blocked unless there is at least one change of the text.

Date

The Date field may specify the date when the note was entered.

Created By

The Created By field may specify an employee name, who created a note.
