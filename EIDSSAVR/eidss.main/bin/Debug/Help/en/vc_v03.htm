Avian Disease Investigation Form (V03)

Fig. 332 Avian Disease Investigation Form

The Avian Disease Investigation Form window consists of:

Case Summary section. Remains displayed when moving through the tabs below. It contains the basic information about an avian case.

Fig. 333 Case Summary section

Field/Variable

Description

Case ID

The field Case ID contains the unique number of the case and will be populated when the case is saved for the first time. The Case ID is generated according to the Unique Numbering Schema.

Field Accession ID

This ID number is an identification that an investigator assigns to the case during filling out the paper form.

Outbreak ID

This attribute is used to link a livestock case to a certain outbreak. An already existing outbreak can be chosen by clicking the button which opens a search form. Also a new outbreak can be entered by clicking the button which opens an outbreak form. Click the button to unlink the case with an outbreak.

Case Status

There are two values: In process and Closed. In process is a default value – it means that the case is still being investigated. If the case is closed and the Closed value is set up, all attributes become inactive.

Diagnoses

String is formed automatically with all clinical diagnoses from Diagnoses tab and can’t be edited.

Case Classification

There are four values: Confirmed, Probable, Refused, Suspect. The default value is null.

–Confirmed is chosen if the case has been confirmed with the epidemiological and the clinical signs, as well as by the lab tests.

–Probable is chosen if the information has not been confirmed clinically, or by laboratory tests.

–Suspect is chosen if the case has been confirmed with the clinical and the epidemiological data, but has not been confirmed by the laboratory.

–Not a case.

Report Type

Report type indicates during which type of surveillance (active or passive) the case was detected. Can be Active or Passive. The field is mandatory. If the case is created within an active surveillance session, the field value is set to Active automatically and becomes non-editable.
Farm Details tab which contains information about the farm - its location, contact information, size and arrangement.

Fig. 334 Farm Details tab

Reported section. The initial report date and a person who reported about a case can be entered here.

 

Field/Variable

Description

Reported By

An already existing person in EIDSS can be chosen here by clicking the button, or create a new person by clicking the button which opens form A09. Click the empty drop-down list item to clear an attribute.

Initial Report Date

Initial report date is less or equal than current date.

Investigated section. Information about the investigator can be entered here; assigned date and investigation date.

 

Field/Variable

Description

Investigator Name

An already existing person in EIDSS can be chosen by clicking button, or create a new person by clicking the button which opens form A09. Click the empty drop-down list item to clear an attribute.

Assigned Date

The date when the case is assigned for investigation.

Investigation Date

When the assigned personnel first interview the farmer and/or collect samples.

Data Entry section. This section has three read-only text-boxes. Information about site, user, and first save date are displayed in it.
 

Field/Variable

Description

Site

Site unique number where the case was first created.

Officer

Name of the user who first saved the case.

Date

Date of first save

Farm Contact Information section. Information about the farm contacts can be entered here. This section links to the farm module. Already existing farms can be found or a new farm can be entered. If an already existing farm is chosen, all the attributes of this section will be filled automatically
 

Field/Variable

Description

Farm ID

Attribute Farm ID displays the unique farm ID number. The attribute has a functional button to search for an existing farm and link it. If a new farm is created, then the ID number will be calculated automatically after the first save of the form.

Farm Name

The farm’s name is entered.

Farm Owner Name

An already existing person in EIDSS can be chosen by clicking the button which opens form H03 Persons List, or enter a new person in the text-box.

Phone

The farm’s phone number is entered.

E-mail

The farm’s e-mail address is entered.

Longitude / Latitude

The farm longitude and latitude can be entered manually or by using the up-down controls, or by the use of the Map button which opens the GIS module.

Longitude is a numerical field. The numbers can be entered in the following format: x.xxxxx and can be entered only in the range of (-180, +180).

Latitude is a numerical field. The numbers are entered in the following format: x.xxxxx and can be entered only in the range of (-85, +85).

When pointing to a location on the map, the Longitude and Latitude are filled automatically.

Country

Name of country where farm is located.

Region

Name of region where farm is located. Only regions of selected country are displayed in drop-down list.

Rayon

Name of rayon where farm is located. Only rayons of selected region are displayed in the drop-down list.

Town or Village

Name of town or village where farm is located. Only towns and villages of selected rayon are displayed in drop-down list.

Street

Name of street where farm is located. Only streets of selected town or village are displayed in the drop-down list. A new street name can be entered in the drop-down list that will be added to the reference list after the case is saved.

Postal code

Postal code of the farm. Only post codes of selected town or village are displayed in the drop-down list. A new postal-code can be entered in drop-down list that will be added to the reference list after the case is saved.

Building/House/Apt.

Building, house, and apartment details of a farm’s location. The sequence of these fields are different for different languages. It is Building/House/Apartment for English and House/Building/Apartment for all other languages

Farm Type section. Farm type can be specified. The section can contain only one attribute.
 

Field/Variable

Description

Farm Type

Farm type is selected.

Farm Details section. Farm details can be specified.
 

Field/Variable

Description

Number of barns/buildings

Number of barns/buildings is entered.

Number of birds per barns/buildings

Number of birds per barns/buildings is entered.
Flock Epi/Clinical Signs tab is designed to describe farm structure comprised of flocks & species, epidemiological information, and clinical signs.

Fig. 335 Flock Epi/Clinical Signs tab

Herd/Species Info section. This section is designed for detailed farm description. It consists of three functional buttons and a table to specify a treelike farm structure.

 

Field/Variable

Description

Farm/Flock/Species

The list of species in drop-down list is restricted to avian species. To add another species from the full reference, click the button – Species reference editor opens. Only one kind of species can be specified for one flock. The species value is mandatory. If a species is created, its value must be selected from the drop-down control.

Total

Total number of birds per species is entered. Totals are calculated automatically.

Dead

Number of dead birds per species is entered. A total number of sick and dead birds cannot be more than the total number of birds. If this occurs, a warning message will appear. Totals are calculated automatically.

Sick

Number of sick birds per species is entered. A total number of sick and dead birds can’t be more than the total number of birds. If this occurs, a warning message will appear. Totals are calculated automatically.

Avg. Age (weeks)

Text comments for affected species are entered.

Start of Signs

Date when signs first appeared is entered.

Buttons

New Flock

To add a flock to the farm structure, click the New Flock button, and a new flock record will be added.

New Species

To add a species to the selected flock click the New Species button, and a new species record will be added. It is active only if a flock is selected.

Delete

To delete select a flock or species, and click the Delete button.

A flock or species can’t be deleted while related vaccinations exist; a warning message will appear: “Can’t delete flock/species. Delete all vaccinations related with this flock/species before”.

Flock or species can’t be deleted while related samples exist; a warning message will appear: “Can’t delete flock/species. Delete all samples related with this flock/species before”.

Delete is active only if a flock or species is selected.

Epidemiological Information/Product Movement/Backward Tracing section. This section is an Avian Farm EPI template and it is displayed when the farm is selected in the Flock/Species Info section.

 

Clear

To clear previously entered values, click Clear. The button is active only if some of attributes are changed.

Copy

Not applicable for this form.

Paste

Not applicable for this form.

Management/Clinical Signs/Necropsy Examination section. This section displays clinical signs of the selected species on the farm. This section is an Avian Species CS template.
 

Clear

To clear previously entered values, click Clear. The button is active only if some of attributes are changed.

Copy

To copy previously clinical signs for a selected species click Copy. All template values will be copied to clipboard. The button is active only if some of attributes are changed.

Paste

To fill clinical signs automatically with copied values from a previous species, click Paste. All copied values will be pasted in the template from clipboard automatically. The button is active only if some attributes are copied.

Samples tab is designed to enter information about collected samples. There are three sections in the tab: Sample Receiving/Registration section - contains list of collected samples, Sample details section – contains the person who collected sample, and Notes section - contains a text field for comments.

Fig. 337 Samples tab

Sample Receiving/Registration section. This section contains a table with information about collected samples. There are two sections in the tab: Sample Receiving/Registration section - contains a list of collected samples, and Additional Test Requested and Sample Notes section - contains text field for any comments

 

Field/Variable

Description

Filter samples by Diagnosis

If Filter by Diagnosis option is enabled, drop-down list of samples is filtered according to the Diagnosis -> Sample Type matrix. The list will be displayed without filtration if the option is disabled.

The default value copies the value of the Filter samples by Diagnosis check-box the the System Prefernces.

Sample Type

Sample type is selected here. Contains the list of values from Sample type reference filtered by HA code Avian.

If Filter by Diagnosis option is enabled, the drop-down list of samples is filtered according to the Diagnosis -> Sample Type matrix.

Field Sample ID

Field sample ID is entered in free-text form.

Species

Species of the animal selected in Animal ID column are displayed.

Bird Status

Status of the received species.

Collection Date

Date when the sample is collected is entered here.

Accession Date

Date when the sample is accessed in a laboratory is displayed here.

Sample Condition Received

The condition in which sample was received by a laboratory is displayed here.

Comment

Additional comments on the sample condition.

Collected by Institution

Institution attribute can be entered that contains a full list of all country organizations. The list of officers will be filled automatically according to the selected organization.

Collected by Officer

When an institution is selected, an already existing officer from Officer attribute can be chosen with the button, or a new officer can be entered with the button which opens form A09.

Sent to Organization

Organization to which sample is sent.

Buttons

Delete

To delete, select a sample, and click the Delete button. A warning message with Yes/No confirmation will appear.

The sample cannot be deleted while there are related penside tests; a warning message will appear: “Can’t delete sample species. Delete all penside tests related with this sample before”.

An accessioned sample cannot be deleted; a warning message will appear: “Sample can’t be deleted”.

 

Additional Test Requested and Sample Notes section. This section contains a memo-box control in which text comments can be entered.

Lab Tests tab is designed to display laboratory tests results and to enter results summary & interpretation. There are two sections: Tests - laboratory tests and tests results are displayed, Results Summary and Interpretation - a summary of results and interpretation are entered here.

 

Fig. 339 Lab Tests tab

Tests section. This section contains a table with laboratory tests, tests results, and an additional space on the right of the table to display specific parameters for the selected test.

 

Field/Variable

Description

Tests Conducted

The field shows if the tests were performed for the given case. If tests were performed in the laboratory and were completed, then Yes is automatically entered to this field and it becomes non-editable. If tests were performed by a lab, but for any reasons the laboratory worker cannot enter results into the system (for example the lab does not have EIDSS installed), select Yes in Tests Conducted field and enter results manually. If by any reasons, detailed information on tests if not entered in a lab and is not required, but it is known that tests were performed, select Yes, and do not enter the results.

Lab Sample ID

Filled automatically with Lab Sample ID. If a derivative or aliquot was created its own lab the ID is displayed.

Sample Type

Filled automatically with Sample Type. If derivative was created, the parent Sample Type is displayed.

Field Sample ID

Filled automatically with the Field Sample ID.

Animal ID/Species ID

Filled automatically with the Animal ID/Species ID.

Test Name

Filled automatically with the Test Name.

Test Diagnosis

Filled automatically with the Test Diagnosis.

Date Tested

Filled automatically with the Date Tested.

Test Category

Filled automatically with the Test Category.

Test Status

Filled automatically with the Test Status.

Result/Observation

Filled automatically with the Test Result.

Amendment History

Displays the information on the test result amendment history if it exists.

Buttons

Delete

To delete, select the test result, and click the Delete button. A warning message with Yes/No confirmation appears.

Note: only manually entered tests can be deleted.

Results Summary and Interpretation section. This section contains a table with laboratory tests, tests results, and a special space on the right of the table to display specific parameters for the selected test.

 

Field/Variable

Description

Diagnosis

Contains the diagnosis for a test which is selected for interpretation. The field is read-only.

Test Name

Filled automatically according to selected test in test table.

Test Category

Filled automatically according to selected test in test table.

Rule Out/ Rule In

This column contains a drop-down list with the Rule in Value for Test Validation reference. The Rule In value can be specified if the test confirms the selected diagnosis. The Rule Out value can be specified, if the test does not confirm the selected diagnosis.

Comments (Rule Out/ Rule In)

Comments about the decision to rule in/out are entered here.

Date Interpreted

Is filled automatically with current date after Rule Out/Rule In value is specified.

Interpreted by

Is filled automatically with current user after Rule Out/Rule In value is specified.

Validated (Y/N)

This column contains a check-box element. By default it has mixed value – the diagnosis is neither declined nor validated by the officer who has permission for validation. If the check-box is selected, then the diagnosis is validated by the officer; if not selected , then the diagnosis is declined.

Comments (Validated)

Comments about validation are entered.

Date Validated

Is filled automatically with current date if Validated (Y/N)=Yes.

Validated by

Is filled automatically with current user if Validated (Y/N)=Yes.

Buttons

New

To add new interpretation, select a test with defined Result in the Tests section and click the New button. New record will be added to the table.

Delete

To delete, select an interpretation, and click the Delete button. A warning message with Yes/No confirmation will appear.

Once saved the interpretation cannot be deleted.
