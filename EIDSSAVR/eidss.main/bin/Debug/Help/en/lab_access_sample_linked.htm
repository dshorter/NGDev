Accession in sample, linked or not linked to case/session (IF)

Samples can be either initially entered in the case/session to which they are linked and later accessioned in the Laboratory module, or added with a simultaneous accession in the Laboratory module. Only samples with an Unaccessioned status can be accessioned in. Samples which have not yet been accessioned in have the Sample Condition Received column blank.

This operation is available only if the Lab Module Simplified Mode check-box is clear in the System Preferences.

 

1.From the Journals menu, select Samples. The Samples List L01 window opens.

2.Click the Accession button; and from the drop-down menu select one of the following depending on which kind of sample shall be accessioned in:

 

•Human Accession In to register a sample linked to a human case. The Human Cases List window opens.

•Vet Accession In to register a sample linked to a veterinary case. The Veterinary Cases List window opens.

•AS Accession In to register a sample linked to an active surveillance session. The Active Surveillance Sessions List window opens.

•VS Accession In to register a sample linked to a vector surveillance session. The Vector Surveillance Sessions List window opens.

 

3.Find and double-click the case or session referring to the sample. The Accession In window opens, its view depends on the type of the selected case or session. The samples already registered for the case/session are displayed in the Sample Receiving/Registration pane. It contains a detailed list of samples linked to the selected case/session.

4.Find a desired sample (select sample's complete or partial ID in the Search by local/field Sample ID field and click the Search button).

5.If the desired sample is not in the list yet (omit this step if the desired sample is already in the list):

a)Click the New button. A new half-empty sample record is added to the Sample Receiving/Registration pane.

b)Enter some basic data required to accession in the sample. For for more details about the fields, read the Accession In L05 section.

Fig. 440 Accession In

6.If the desired sample is in the list, select it and click the Accession button. The sample becomes editable, the Accession Date cell is filled in automatically with the current date and can be edited.

7.Change the Sample Condition Received value and, if so, enter an explanation to the Comment cell. If the Sample Condition Received field value is changed to Rejected and saved, the sample is removed and cannot be found by search.

Now test can be assigned without leaving the Accession In window, if necessary. For details, read the Assign tests during sample accession in section.

8.Print barcode labels for newly accessioned in samples.

9.Save and close the Accession In window.

The sample is accessioned in. The changed made in the case/session are replicated to other EIDSS sites in accordance with the filtration rules.
