Vector Surveillance Session W02

The Vector Surveillance Session (W02) is a form used to enter, view, or edit general, detailed and summary information about a VS session.

Information about a vector surveillance session can be registered in details or summarized. Detailed information is comprised of pools/vectors, collected during the session, the area and time of their collection, and the name of the person who collected the material. Once the information about collected material is registered, the samples obtained from the collected material are defined. And the latter steps are registration of tests, first field tests and then laboratory tests performed for the collected material. All steps mentioned above are performed in the Vector Surveillance module except for the laboratory tests, which are conducted in the Laboratory module and only displayed in the Vector Surveillance module.

All samples and samples registered in the Vector Surveillance module are available in the Laboratory module.

Summarized information is a very brief information containing data about collected vectors, the location where they were collected and diseases confirmed by tests performed for the collected material. The information about tests: number, type, and date are not registered; only the confirmed diseases are registered.

Fig. 398 Vector Surveillance Session form

The Vector Surveillance Session form is comprised of the following:

•General Information pane

•Detailed Info tab with subtabs:

•Pools/Individual Vectors subtab

•Samples subtab

•Fields Tests subtab

•Lab Tests subtab

•Summary Info tab

Buttons

Save

Saves the changes made in the open session. The Confirmation window opens asking to confirm saving.

OK

Closes the vector Surveillance session window. If the changes made in the session are not saved, the Confirmation window opens asking to confirm saving.

Cancel

Cancels all unsaved changes. Once the button is clicked, the Confirmation window opens asking to confirm changes cancellation.

Delete

Deletes the open session. Only an active session, i.e., the session with the In Progress status, that has neither vectors nor outbreaks connected to it can be deleted.
