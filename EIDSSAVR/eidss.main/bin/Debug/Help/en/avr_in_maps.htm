Maps in AVR

The data received and displayed in the view table can be drawn in a map. The EIDSS map can represent a number of objects (usually cases or outbreaks) for a region and rayons as charts or color intensity. The map often makes it easier to understand the data in a worksheet, because users can easily pick out patterns and trends illustrated in the chart on a map or marked with a color that are otherwise difficult to see.

Before the map preparation is started, decide what kind of map is required. What data needs to be displayed in charts, and which one is to be displayed as a color. The data for the charts and color gradient has to be placed to the data area of the layout. The geographical units need to be placed to the row area. If both rayons and regions are used in a pivot grid, and regions are planned to be used in the map creation, use Row totals. For details about using totals, read the Show totals section.
