Elements of the Livestock Disease Investigation Form window:

Case Summary section remains displayed when moving through the tabs below. It contains the basic information about the livestock case.

Fig. 323 Case Summary section

Field/Variable

Description

Case ID

The field Case ID contains the unique number of the case and will be populated when the case is saved for the first time. The Case ID is generated according to the Unique Numbering Schema.

Field Accession ID

This ID number is the identification that an investigator assigns to the case during paper form filling.

Outbreak ID

This attribute is used to link a livestock case to a certain outbreak. An already existing outbreak can be chosen by clicking button (search form will be opened) or enter a new outbreak by clicking button (outbreak form will be opened). Click the button to unlink the case with an outbreak.

Case Status

There are two values: In process and Closed. In process is a default value defining that the case is still being investigated. If case is closed and the Closed value is set up, then all attributes become inactive.

Diagnoses

A string field is formed automatically with all clinical diagnoses from the Diagnoses tab and can’t be edited. Diagnoses are separated by <,> sign in the string.

Case Classification

There are three values: Confirmed, Probable and Suspect. Default value is null.

–Confirmed is chosen if the case has been confirmed with the epidemiological and the clinical signs, as well as by the lab tests.

–Probable is chosen if the information has not been confirmed clinically, or by laboratory tests.

–Suspect is chosen if the case has been confirmed with the clinical and the epidemiological data, but has not been confirmed by the laboratory.

–Not a Case is chosen if case has not been confirmed.

Report Type

Report type indicates during which type of surveillance (active or passive) the case was detected. Can be Active or Passive. The field is mandatory. If the case is created within an active surveillance session, the field value is set to Active automatically and becomes non-editable.

Session ID

Session ID if the case was created within an active surveillance session. An already existing session can be opened by clicking the button.

Description

Site

Site unique number where the case was first created.

Name of the user who first saved the case.

Farm Contact Information section. Information about farm contacts can be inputted here. Already existing farms can be found here or new farms can be entered.
 

Field/Variable

Description

Farm ID

Attribute Farm ID displays unique farm ID number. Attribute has functional button to search for existing farm and link it. If user creates a new farm, the case ID number will be calculated automatically after the first save of the form.

Farm Name

The farm’s name is entered here.

Farm Owner Name

An already existing person in EIDSS can be chosen by clicking button (form H03 Persons List opens) or enter a new one in the text-box (Last Name, First Name, Middle Name values).

Phone

Farm’s phone number.

E-mail

Farm’s e-mail address.

Longitude / Latitude

The farm's longitude and latitude is entered manually or by using up-down controls, or by the use of Map button (GIS module will be opened).

Longitude is a numerical field. The numbers can be entered in the following format: x.xxxxx and can be entered only in the range of (-180, +180).

Latitude is a numerical field. The numbers are entered in the following format: x.xxxxx and can be entered only in the range of (-85, +85).

If a location is point to on the map, Longitude and Latitude are filled automatically.

Country

Country where farm is located. The field is mandatory.

Region

Region where farm is located. Only regions of selected country are displayed in drop-down list. The field is mandatory.

Rayon

Rayon where farm is located. Only rayons of selected region are displayed in drop-down list. The field is mandatory.

Town or Village

Town or village name where farm is located. Only towns and villages of selected rayon are displayed in drop-down list.

Street

User can enter here street where farm is located. Only streets of selected town or village are displayed in drop-down list. User can enter new street name in drop-down list that will be added to the reference after case save.

Postal code

Postal code of the farm. Only post codes of selected town or village are displayed in drop-down list. User can enter new postal-code in drop-down list that will be added to the reference after case save.

Building/House/Apt.

Building, house and apartment details of farm’s location. The sequence of these fields are different for different languages. It is Building/House/Apartment for English and House/Building/Apartment for all other languages.

Ownership structure Section. User can specify farm ownership structure. Section contains only one attribute.
 

Ownership structure

Ownership structure is specified here.
Herd Epi/Clinical Signs/Control Measures tab is designed to describe farm structure (herds, species), epidemiological information, clinical signs and control measures.

Fig. 325 Herd Epi/Clinical Signs/Control Measures tab

Herd/Species Info section. This section is designed for detailed farm description. It consists of three functional buttons and a table to specify treelike farm structure.

 

Field/Variable

Description

Farm/Herd/Species

Select the species from the drop-down list or click button to add another species to the Species reference editor. One species can be specified for one herd only once, but the herd can hold different kind of species. The Species value is mandatory.

Total

Column Total is designed to enter total number of animals of certain species in the herd at the time of the report. To enter the number of animals, use the cell in Species row. This cell has an integer type and displays like a text-box. The number of species in every herd and farm will be calculated automatically.

Dead

The number of dead animals from the total is entered in the Dead column. To enter the number of dead animals, use certain cell in the Species row. This cell has integer type and displays like an up-down control. The number of dead animals can’t be more than the total number. The number of dead species in every herd and farm will be calculated automatically.

Sick

The number of sick animals from the total is entered in the Sick column. To enter the number of sick animals, use certain cell in Species row. This cell has an integer type and displays like an up-down control. The number of sick animals can’t be more than the total number. The number of sick species in every herd and farm will be calculated automatically.

Start of Signs

Date when the first signs of sickness were detected.

Note (Include breed)

Column Note is designed to enter comments for species. It is a string attribute with a memo-box control.

Buttons

New Herd

Clicking on the New Heard button adds a herd which creates a new heard in the farm structure.

New Species

To add an additional species, select a herd row in the grid and click New Species button. A new species record will be added for the selected herd.

This button is active only if the herd or species row selected.

Delete

To delete select a herd or species, and click the Delete button.

The herd or species can’t be deleted while there are related animals, warning message will appear: "Can’t delete herd/species. Delete all animals related with this herd/species before".

Herd or species can’t be deleted while there are related vaccinations, warning message will appear: “Can’t delete herd/species. Delete all vaccinations related with this herd/species before”.

This button is active only if herd or species row selected.

 

Epidemiological Information section. This section is designed for farm epidemiological situation observation and it is displayed when the farm is selected in the Herd/Species Info section. This is the Livestock Farm EPI template, and it can be changed in the Flexible Form Designer.

 

Clear

To clear all earlier entered values click Clear. The button is active only if some of attributes are changed.

Copy

Not applicable for this form.

Paste

Not applicable for this form.

Species epidemiological and clinical investigation section. This section displays clinical signs of the selected species on the farm and. The section is displayed only if the Species is defined in the Herd/Species Info section. This is the Livestock Species CS template and it can be changed in Flexible Form Designer.
 

Clear

To clear all earlier entered values click Clear. The button is active only if some of attributes are changed.

Copy

To copy all entered clinical signs for selected species click Copy, all template values will be copied to the clipboard. The button is active only if some of attributes are changed.

Paste

Click Paste to fill clinical signs automatically with copied values from previous species. All copied values will be automatically pasted in the template from the clipboard. Paste is active only if some attributes are copied.

Control measures section. Displays preventive measures taken on the farm. This section is the Livestock Control Measures template. It is displayed under the Herd/Species Info table, and it can be changed in the Flexible Form Designer.

Animals tab is designed to enter information and clinical signs of affected animals from which samples were collected at the time of the investigation.

Fig. 327 Animals tab

Field/Variable

Description

Herd ID

Drop-down list which contains a list of all herds of the farm that are entered in the Herd Epi/Clinical Signs/Control Measures tab. This field is mandatory field.

Animal ID

Text-box for animal identification. This ID number can be formed automatically according to the numbering schema, or can be entered manually.

Species

Drop-down list containing all of a herd’s species.

Age

Drop-down list with Animal Age reference filtered according to the Species -> Animal Age Matrix.

Sex

Drop-down list with Animal Sex reference.

Clinical Signs

A drop-down form to enter clinical signs for the added animal. Livestock Animal CS template.

All previously entered clinical signs can be cleared by clicking the Clear button, and all entered clinical signs for another animal can be copied and pasted with the Copy and Paste buttons.

Clear and Copy buttons are active only if some of the attributes are changed. The Paste button is active only if some attributes are copied.

Status

Drop-down list with Animal Condition reference.

Buttons

Delete

To delete a record, select it and click the Delete button.

An animal cannot be deleted if it has accessed samples, warning message will appear: “This animal can’t be deleted”.

 

Samples tab is designed to enter information about collected samples. There are two sections in the tab: Sample Receiving/Registration section - contains a list of collected samples, and Additional Test Requested and Sample Notes section - contains text field for any comments.

Fig. 328 Samples tab

Sample Receiving/Registration section. This section contains a table with information about collected samples.

 

Field/Variable

Description

Filter samples by Diagnosis

If the Filter by Diagnosis option is enabled, drop-down list of samples will be filtered according to the Diagnosis -> Sample Type matrix.

If the option is disabled, the list will be displayed without regarding to the diagnosis.

The default value copies the value of the Filter samples by Diagnosis check-box the the System Prefernces.

Sample Type

Sample type is selected in this field. The field contains the list of values from the Sample type reference filtered by HA code Livestock.

If the Filter by Diagnosis option is enabled, a drop-down list of samples is filtered according to the Diagnosis -> Sample Type matrix.

Field Sample ID

Field sample ID is entered in a free-text form.

Animal ID

Contains the list of all animals entered in the Animals table. A drop-down list contains three columns: Animal ID, Species and Sex.

Only Animal ID value is displayed in the field.

Species

Species of the animal selected in Animal ID column are displayed.

Collection Date

Date when the sample is collected is entered here.

Accession Date

Date when the sample is accessed in a laboratory is displayed here.

Sample Condition Received

The condition in which sample was received by a laboratory is displayed here.

Comment

Additional comments on the sample condition.

Collected by Institution

The Institution attribute that contains a full list of all country organizations – the list of officers will be filled automatically according to the selected organization.

Collected by Officer

When an institution is selected, an already existing officer can be selected from the Officer attribute with button or enter new one with button (form A09 will be opened).

Sent to Organization

Organization to which the sample is sent.

Buttons

Delete

To delete a sample, select it and click Delete button. A warning message with Yes/No confirmation will appear.

A sample can’t be deleted while there are related penside tests. A warning message will appear: “Can’t delete sample species. Delete all penside tests related with this sample before”.

An accessioned sample can’t be deleted. A warning message will appear: “Sample can’t be deleted”.

 

Additional Test Requested and Sample Notes section. This section contains memo-box control in which text comments can be specified.

Lab tests tab is designed to display laboratory tests results, and to enter results summary and interpretation. There are two sections: Tests - laboratory tests and tests results are displayed, Results Summary and Interpretation - results summary and interpretations are entered here.

 

Fig. 330 Lab Tests tab

Tests section. This section contains a table with laboratory tests, tests results, and special space on the right of the table to display specific parameters for the selected test.

 

Field/Variable

Description

Tests Conducted

The field shows if the tests were performed for the given case. If tests were performed in the laboratory and were completed, then Yes is automatically entered to this field and it becomes non-editable. If tests were performed by a lab, but for any reasons the laboratory worker cannot enter results into the system (for example the lab does not have EIDSS installed), select Yes in Tests Conducted field and enter results manually. If by any reasons, detailed information on tests if not entered in a lab and is not required, but it is known that tests were performed, select Yes, and do not enter the results.

Lab Sample ID

Filled automatically with Lab Sample ID. If derivative or aliquot was created, its lab ID will be displayed.

Sample Type

Filled automatically with Sample Type. If derivative was created, parent Sample Type is displayed.

Field Sample ID

Field Sample ID of the sample, which is created and saved on the Samples tab.

Animal ID/Species ID

ID of the Animal, associated with the sample.

Test Name

Test name is selected from drop-down list. List of tests is filtered by HA code.

Test Diagnosis

Test Diagnosis is selected form drop-down list. List contains values of all tentative and final diagnoses form Diagnoses tab.

Date Tested

Date when the chosen test was conducted.

Test Category

Test Category is selected from drop-down list.

Test Status

The value is auto populated with Completed.

Result/Observation

Test result is selected from drop-down list.

Amendment History

Displays the information on the test result amendment history if it exists.

Buttons

Delete

To delete test result, select it and click the Delete button. Warning message with Yes/No confirmation will appear.

Note: only manually entered tests can be deleted.

Results Summary and Interpretation section. This section contains a table with laboratory tests, tests results and special space on the right of the table to display specific parameters for the selected test.

 

Field/Variable

Description

Diagnosis

Contains the diagnosis for a test which is selected for interpretation. The field is read-only.

Test Name

Filled automatically according to selected test in test table.

Test Category

Filled automatically according to selected test in test table.

Rule Out/Rule In

This column contains a drop-down list with Rule in Value for Test Validation reference. Rule In value can be specified, if the test confirms selected diagnosis. Rule Out value can be specified, if the test does not confirm selected diagnosis.

Comments (Rule Out/Rule In)

Comments about the decision to rule in/out are entered here.

Date Interpreted

Is filled automatically with current date after Rule Out/Rule In value is specified.

Interpreted by

Is filled automatically with current user after Rule Out/Rule In value is specified.

Validated (Y/N)

This column contains a check-box element. By default it has a mixed value – it means that the diagnosis is neither declined nor validated by an officer who has permission for validation. If the check-box is selected it means that diagnosis is validated by officer, if unselected – declined.

Comments (Validated)

Comments about validation are entered here.

Date Validated

Is filled automatically with current date if Validated (Y/N)=Yes.

Validated by

Is filled automatically with current user if Validated (Y/N)=Yes.

Buttons

New

To add new interpretation, select a test with defined Result in the Tests section and click the New button. New record will be added to the table.

Delete

To delete interpretation, select it and click the Delete button. A warning message with a Yes/No confirmation will appear.

Once saved, the interpretation can’t be deleted.
