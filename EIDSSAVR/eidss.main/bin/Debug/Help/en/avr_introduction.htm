Introduction to AVR

The Analysis, Visualization and Reporting (AVR) module provides users with various capabilities to investigate and display the data collected in the system. This means that users can get access to any variable in the database and explore human, veterinary, vector surveillance, and lab data all together.

The work with the AVR module generally consists of two steps: building a query and creating a layout based on of the query.

Query building is a preparatory stage required to create a database request, which is aimed to extract the data to be analyzed in the final report. When building a query, the user has to indicate objects and their attributes to be extracted from the database. The query configuration, its objects and attributes define the types of the reports that can be configured on it, and rate the speed of its processing. To increase the speed of the reports' processing, special filters can be used that limit the amount of information to be extracted from the database, and thereby accelerate the report display. For more details, see the Queries section.

The layout provides the user with an opportunity to develop the report he/she needs based on the data extracted with the query from the database. The layout is designed to group the received information,  display it in the view required by the user, and adapt it to the user's specification by special functions or filters. In its turn, the work with a layout shall consist of two stages: pivot grid creation and data view definition. For more details, see the Layouts section.
