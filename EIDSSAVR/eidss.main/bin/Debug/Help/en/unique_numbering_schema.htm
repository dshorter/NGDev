Unique Numbering Schema

Show/Hide Hidden Text

EIDSS contains various sets of items, such as disease cases, patients, samples, tests, etc. To uniquely identify an item, EIDSS uses a unique numbering schema. In some cases the user is able to assign an ID to an item, and the other IDs are assigned automatically according to the numbering schema. Automatic numbering ensures assigning items unique IDs, and thus prevents any two items from having the same ID. Moreover the number of an item identifies the year of the number association, site, type of the item (sample, case or other), etc. Defining a numbering schema for an item means defining the string of symbol groups making up the ID.

Set up the numbering before using EIDSS, and do not change it! Numbering schema change can cause serious database damage.

Open the list of unique numbering schemas:

In the System menu, click Configuration, then click Unique Numbering Schema. The Unique Numbering Schema List window opens.

Fig. 91 The Unique Numbering Schema List window

 

Edit the unique numbering schema:

1.Select the ID Type, for which the numbering schema shall be edited and click the Edit button. The Unique Numbering Schema Details window opens.

Fig. 92 The Unique Numbering Schema window

The Number Format section defines the Numbering: in the Prefix field, enter the prefix for the current ID type. Ensure the prefix differs from the other type ID prefixes. In the Next Number Value field, enter the counter. The Preview field displays the result of the numbering set-up.

2.Save the changes and close the window.
